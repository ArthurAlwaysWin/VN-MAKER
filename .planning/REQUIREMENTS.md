# v0.6 Requirements — 主题包系统

## Token Foundation (Token 基础)

- [ ] **TKN-01**: 用户可通过 ~35-40 个设计 Token（颜色/字体/圆角/透明度）控制游戏全 UI 外观，所有 token 以 `--gm-*` CSS 自定义属性渲染
- [ ] **TKN-02**: 用户可选择两个字体槽位（标题/显示字体 + 正文/UI 字体），应用到游戏所有 UI 文字
- [ ] **TKN-03**: 用户可控制全局圆角值（从直角到大圆角），一个 token 影响所有面板/按钮/对话框
- [ ] **TKN-04**: 所有面板/覆盖层背景使用统一 panel-bg token，替代当前 6 处不同的硬编码半透明背景
- [ ] **TKN-05**: 所有按钮（快捷栏/菜单/存读档/设置页）使用统一按钮 token 组（背景色/文字色/悬停色/按下色）
- [ ] **TKN-06**: 未应用主题时，游戏 UI 外观与当前版本完全一致（零视觉回归）

## 9-Slice Image System (九宫格图片系统)

- [ ] **9SL-01**: 用户可为对话框设置九宫格背景图片，替代纯色/渐变背景，图片自动拉伸适配对话框尺寸
- [ ] **9SL-02**: 用户可为面板背景（菜单/存读档界面/设置覆盖层）设置九宫格图片
- [ ] **9SL-03**: 用户可为按钮设置九宫格图片，支持正常/悬停/按下三种状态分别指定不同图片
- [ ] **9SL-04**: 九宫格图片与圆角样式共存（通过 ::before 伪元素实现，不互斥）

## Color Harmony (配色和谐)

- [ ] **CLR-01**: 用户选择一个主色后，系统自动生成完整协调配色方案（强调色/背景色/文字色/边框色）
- [ ] **CLR-02**: 提供多种配色算法供选择（互补色/类似色/三角色/分裂互补色）
- [ ] **CLR-03**: 生成的配色方案经过 WCAG 对比度验证，暗色主题下保证文字可读性（≥4.5:1 正常文字，≥3:1 大文字）

## Theme Editor (主题编辑器)

- [ ] **EDT-01**: 编辑器新增「🎨 主题」标签页，提供可视化主题编辑界面
- [ ] **EDT-02**: 主题编辑器包含颜色选择器、字体选择器、圆角滑块、透明度滑块等控件
- [ ] **EDT-03**: 主题编辑器包含九宫格图片上传和切片参数配置界面
- [ ] **EDT-04**: 主题编辑器内嵌引擎预览 iframe，修改任何 token 立即看到游戏 UI 变化（实时预览）
- [ ] **EDT-05**: 主题编辑器包含配色方案生成器，选主色 → 预览完整调色盘 → 一键应用到 token

## Theme Presets (主题预设)

- [ ] **PRE-01**: 内置 3-4 套精品主题预设（现代/和风/幻想/简约），用户一键应用即获得专业美术风格
- [ ] **PRE-02**: 用户可在预设基础上微调任意 token，而非只能整体覆盖

## Theme Pack (主题包)

- [ ] **PKG-01**: 用户可将当前主题导出为 .theme 文件（ZIP 包，含 token JSON + 引用的九宫格图片资源）
- [ ] **PKG-02**: 用户可导入 .theme 文件，自动提取图片到项目资源目录并应用 token
- [ ] **PKG-03**: 主题文件包含 formatVersion 字段，确保未来版本前向兼容

## Engine Integration (引擎集成)

- [ ] **ENG-01**: ThemeManager 模块读取 script.json 中的 ui.theme 数据，启动时自动注入 CSS 自定义属性
- [ ] **ENG-02**: 主题数据存储在 script.json 的 ui.theme 字段（与 ui.titleScreen/ui.settingsScreen 一致），享有自动保存和撤销/重做
- [ ] **ENG-03**: 用户可一键重置主题为默认值，立即恢复游戏原始外观

## Future Requirements (推迟到后续)

- [ ] 每组件独立样式覆盖（对话框用不同主色 vs 菜单）— v0.7
- [ ] 每场景/每页面主题切换 — v0.7+
- [ ] 布局变体（每个界面 2-3 种预设布局选择）— v0.7
- [ ] 主题包字体内嵌 — v0.7+（字体体积大 + 许可证问题）
- [ ] 动画/转场主题化 — v0.7+
- [ ] 社区主题市场 — v0.8+
- [ ] 更多内置预设（8-10 套覆盖不同题材）— 持续补充

## Out of Scope

- 自由拖拽布局（每个 UI 界面都支持画布设计器） — 工程量过大，且大多数用户不需要
- CSS 编辑器（直接编辑 CSS 代码） — 违反"不碰逻辑"核心理念
- 运行时动态主题切换（游戏中按钮切换主题） — 主题是项目级设置，非玩家设置
