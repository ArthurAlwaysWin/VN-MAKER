**A. GameMenu.js — Don't hide menu when opening settings (D-08):**

In the click event handler (around line 55-69), the current code calls `this.hide()` unconditionally on line 60 before the switch statement. Change it to skip hiding for the 'settings' action.

Replace lines 55-69 (the entire click event listener body):

```js
this.el.addEventListener('click', (e) => {
  const btn = e.target.closest('[data-action]');
  if (!btn) return;
  const action = btn.dataset.action;

  // D-08: Don't hide game menu when opening settings — overlay stacks on top
  if (action !== 'settings') {
    this.hide();
  }

  switch (action) {
    case 'save': if (this.onSave) this.onSave(); break;
    case 'load': if (this.onLoad) this.onLoad(); break;
    case 'backlog': if (this.onBacklog) this.onBacklog(); break;
    case 'settings': if (this.onSettings) this.onSettings(); break;
    case 'title': if (this.onTitle) this.onTitle(); break;
    case 'close': break; // already hidden
  }
});
```

The only change is: `this.hide()` (unconditional) → `if (action !== 'settings') { this.hide(); }`.

**B. main.js — ESC key priority for settings overlay (D-09):**

The current keyboard handler (lines 288-315) has `if (!isPlaying) return;` as the first line, which blocks ALL keyboard handling when on the title screen. But settings can be opened from the title screen (line 429-431), so ESC must work there too.

Replace the entire `keydown` event listener (lines 288-315) with:

```js
document.addEventListener('keydown', (e) => {
  // D-09: Settings overlay ESC works regardless of play state
  if (e.key === 'Escape' && settingsScreen.isVisible) {
    settingsScreen.hide();
    return;
  }

  if (!isPlaying) return;

  switch (e.key) {
    case 'Escape':
      gameMenu.toggle();
      break;
    case ' ':
    case 'Enter':
      e.preventDefault();
      if (engine.waiting && !engine.ended) {
        dialogueBox._handleClick();
      }
      break;
    case 'a':
    case 'A':
      toggleAuto();
      break;
    case 's':
    case 'S':
      toggleSkip();
      break;
    case 'l':
    case 'L':
      gameMenu.onBacklog();
      break;
  }
});
```

The ONLY change is: added 4 lines at the top of the handler (before the `if (!isPlaying) return;` guard) that check `settingsScreen.isVisible` and call `settingsScreen.hide()` if ESC is pressed while settings is open. This gives settings overlay ESC priority over game menu toggle (D-09) and works even from the title screen where `isPlaying === false`.

**Note on D-07 (stack-based layer management):** When settings is closed after being opened from game menu, `GameMenu` was never hidden (per D-08), so it naturally reappears. When opened from title screen, title screen was never hidden either. No extra "return to previous layer" logic is needed — the stack behavior is automatic.
