**1. Add `createDefaultPage()` internal utility** (NOT exported — used internally by addPage/addScene):
```javascript
function createDefaultPage() {
  return {
    id: 'p' + Date.now() + '-' + Math.random().toString(36).substr(2, 4),
    type: 'normal',
    background: null,
    characters: [],
    bgm: null,
    se: null,
    dialogues: [{ speaker: null, text: '', expression: null }],
    transition: { type: 'fade', duration: 800 },
  };
}
```
Use timestamp+random for IDs per Research Pitfall 1 — NEVER use sequential `p1`, `p2` for new pages (demo data uses simple IDs but editor must not).

**2. Add these store functions** (all call `pushState()` immediately — discrete operations per Research Pitfall 4):

- `addScene(sceneId, sceneName)` — Create `data.value.scenes[sceneId] = { name: sceneName, pages: [createDefaultPage()] }`. Guard: return if `!data.value` or if sceneId already exists.
- `deleteScene(sceneId)` — `delete data.value.scenes[sceneId]`. Guard: return if scene doesn't exist.
- `renameScene(sceneId, newName)` — Set `data.value.scenes[sceneId].name = newName`. Guard: return if scene doesn't exist.
- `addPage(sceneId, afterIndex = -1)` — Create page with `createDefaultPage()`. Insert into `scene.pages` at `afterIndex + 1` (or append if -1) using `splice`. Return `{ page, index }` for caller to select. Guard: return null if scene doesn't exist.
- `deletePage(sceneId, pageIndex)` — Remove page at index using `splice(pageIndex, 1)`. Guard: return if invalid index.
- `reorderPages(sceneId, fromIndex, toIndex)` — Splice out page from `fromIndex`, splice into `toIndex`. Guard: return if scene doesn't exist or indices out of bounds.

**3. Update the return statement** to include all 6 new functions:
```javascript
return {
  data, isLoading, _skipWatch,
  pushState, undo, redo, historyIndex, history,
  loadFromData, reset,
  getSettingsScreen, updateSettingsScreen,
  getTitleScreen, updateTitleScreen,
  addScene, deleteScene, renameScene,
  addPage, deletePage, reorderPages,
  loadScript, saveScript
};
```

**Undo/redo note (Pitfall 4):** All 6 functions call `pushState()` at the end because they are discrete user actions. Do NOT add pushState to continuous operations (dragging, typing) — App.vue's 500ms debounced watcher handles those.

**File 1: `src/editor/composables/usePageEditor.js`** — Create new file.

This composable provides shared selection state for the page editor. Uses Vue provide/inject so all child components (SceneTree, PageCanvas, PageInspector) share the same reactive state without prop drilling.

```javascript
import { ref, computed, provide, inject } from 'vue';
import { useScriptStore } from '../stores/script.js';

const PAGE_EDITOR_KEY = Symbol('pageEditor');
```

**`createPageEditor()` function** (called once in PageEditor.vue setup):
- Initialize `useScriptStore()` into local `script` const
- Create refs: `selectedSceneId = ref(null)`, `selectedPageIndex = ref(0)`, `selectedDialogueIndex = ref(0)`, `selectedCharIndex = ref(-1)`
- Create `showCharPicker = ref(false)` — controls character picker modal visibility
- Create computeds:
  - `currentScene` — returns `script.data.scenes[selectedSceneId.value]` or null
  - `currentPage` — returns `currentScene.value?.pages?.[selectedPageIndex.value]` or null
  - `currentDialogue` — returns `currentPage.value?.dialogues?.[selectedDialogueIndex.value]` or null
- Create methods:
  - `selectPage(sceneId, pageIndex)` — set selectedSceneId, selectedPageIndex, reset dialogue/char index to 0/-1
  - `selectDialogue(index)` — set selectedDialogueIndex
  - `selectCharacter(index)` — set selectedCharIndex
  - `initSelection()` — auto-select first scene/page from `Object.entries(script.data.scenes)` if available. Call this on mount.
- Assemble all into an object, call `provide(PAGE_EDITOR_KEY, editor)`, return the object

**`usePageEditor()` function** (called in child components):
- `const editor = inject(PAGE_EDITOR_KEY)`
- If `!editor` throw `new Error('usePageEditor() must be used inside PageEditor')`
- Return editor

Export both functions as named exports (no default export per project conventions).
