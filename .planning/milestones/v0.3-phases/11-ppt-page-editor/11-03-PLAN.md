WYSIWYG canvas implementing EDITOR-05 (1280×720 canvas), EDITOR-06 (character positioning), and D-07 (inline dialogue editing).

This component adapts the canvas rendering pattern from CanvasPreview.vue but reads from page data directly (NOT command replay — see Research Pattern 2).

Template structure:
```html
<div class="canvas-wrapper" ref="wrapperRef" @click="onCanvasClick">
  <div class="canvas-artboard" :style="artboardStyle">
    <!-- Background -->
    <div class="canvas-bg" :style="bgStyle"></div>

    <!-- Characters (DraggableElement per character in page.characters[]) -->

      :key="char.id + '-' + idx"
      :x="getCharX(char)"
      :y="getCharY(char)"
      :scale="char.scale || 1"

      :canvas-scale="canvasScale"

        </div>
      </div>
```

Script setup:

1. **Imports:** `ref, computed, onMounted, onBeforeUnmount, nextTick, watch` from vue; `DraggableElement` from `../canvas/DraggableElement.vue`; `usePageEditor` from composable.

2. **State:**
   ```javascript
   const editor = usePageEditor();
   const { script } = editor;
   const wrapperRef = ref(null);
   const canvasScale = ref(1);
   const isEditingDialogue = ref(false);
   const inlineTextarea = ref(null);

   const GAME_W = 1280;
   const GAME_H = 720;
   const PRESET_X = { left: 200, center: 640, right: 1080 };
   const PRESET_Y = 200;
   ```

3. **Computed convenience** (null-safe):
   ```javascript
   const page = computed(() => editor.currentPage.value);
   const currentDialogue = computed(() => editor.currentDialogue.value);
   ```

4. **Canvas scaling** — copy the exact ResizeObserver pattern from CanvasPreview.vue:
   - `updateScale()` reads wrapper bounding rect, computes `Math.min(w/GAME_W, h/GAME_H, 1)`
   - `onMounted` → `updateScale()` + `new ResizeObserver(updateScale).observe(wrapperRef)`
   - `onBeforeUnmount` → `resizeObserver.disconnect()`
   - `artboardStyle` computed → `{ width: GAME_W+'px', height: GAME_H+'px', transform: scale(canvasScale), transformOrigin: 'top left' }`

5. **Background rendering:**
   ```javascript
   const bgStyle = computed(() => {
     const bg = page.value?.background;
     if (!bg) return { background: '#1a1a2e' };
     return {
       backgroundImage: `url("${resolveAsset(bg)}")`,
       backgroundSize: 'cover',
       backgroundPosition: 'center',
     };
   });
   function resolveAsset(path) {
     if (!path) return '';
     if (path.startsWith('asset://') || path.startsWith('http')) return path;
     return `asset://${path}`;
   }
   ```

6. **Character helpers:**
   - `getCharX(char)` → returns `char.x != null ? char.x : (PRESET_X[char.position] || PRESET_X.center)`
   - `getCharY(char)` → returns `char.y != null ? char.y : PRESET_Y`
   - `getCharImage(char)` → look up `script.data.characters[char.id]?.expressions?.[char.expression]`, resolveAsset if found, else null
   - `getCharName(charId)` → `script.data.characters[charId]?.name || charId`

7. **Character drag** (EDITOR-06, D-06):
   ```javascript
   function onCharMove(charIndex, { x, y }) {
     const char = page.value?.characters?.[charIndex];
     if (!char) return;
     char.x = x;
     char.y = y;
     char.position = 'custom';
     // Do NOT call pushState here — continuous drag operation (Pitfall 4)
     // App.vue 500ms debounced watcher handles undo snapshots
   }
   ```

8. **Canvas click deselection:**
   ```javascript
   function onCanvasClick() {
     editor.selectCharacter(-1);
     if (isEditingDialogue.value) stopInlineEdit();
   }
   ```

9. **Inline dialogue editing** (D-07):
   - `startInlineEdit()` → set `isEditingDialogue = true`, `nextTick` → focus `inlineTextarea.value`
   - `onInlineTextInput(value)` → `currentDialogue.text = value` (direct reactive mutation, NO pushState — continuous typing per Pitfall 4)
   - `stopInlineEdit()` → set `isEditingDialogue = false`
   - `onDialogueClick()` → just stops propagation (prevents canvas click deselect)

10. **Dialogue box style:**
    ```javascript
    const dialogueBoxStyle = computed(() => ({
      position: 'absolute',
      bottom: '0',
      left: '0',
      right: '0',
    ```

   - `getCharExpressions(charId)` → `script.data.characters[charId]?.expressions || {}`
   - `truncate(text, len)` → `text?.length > len ? text.slice(0, len) + '…' : text || ''`

4. **Page property setters** (discrete operations — call pushState):
   - `setTransitionType(type)` → ensure `page.value.transition` exists, set `.type`, `script.pushState()`
   - `setTransitionDuration(val)` → parse int, set `.duration`, `script.pushState()`

5. **Character management** (EDITOR-06):
   - `setCharExpression(idx, expr)` → `page.value.characters[idx].expression = expr`, `script.pushState()`
   - `removeCharacter(idx)` → `page.value.characters.splice(idx, 1)`, `script.pushState()`. If removed char was selected, `editor.selectCharacter(-1)`.

6. **Dialogue management** (EDITOR-07, D-07, D-08):
   - `addDialogue()` → push `{ speaker: null, text: '', expression: null }` to `page.value.dialogues`, `script.pushState()`. Select the new dialogue.
   - `removeDialogue(idx)` → splice. `script.pushState()`. Adjust selectedDialogueIndex if needed.
   - `setSpeaker(charId)` → `selectedDialogue.value.speaker = charId || null`, `script.pushState()`
   - `setDialogueExpression(expr)` → `selectedDialogue.value.expression = expr || null`, `script.pushState()`
   - `setDialogueText(text)` → `selectedDialogue.value.text = text` — do NOT call pushState (continuous typing, Pitfall 4). Let App.vue debounced watcher handle it.

7. **Dialogue drag-reorder** (within inspector list):
   - `onDlgDragStart(e, idx)` → set `dlgDragState.fromIndex = idx`, `e.dataTransfer.effectAllowed = 'move'`, `e.dataTransfer.setData('text/plain', String(idx))`
   - `onDlgDragOver(e, idx)` → `e.preventDefault()`
   - `onDlgDrop(e, idx)` → reorder dialogues in `page.value.dialogues` using splice (same pattern as page reorder). `script.pushState()`. Update selectedDialogueIndex to new position.
   - `onDlgDragEnd()` → reset dlgDragState.

8. **Audio setters** (EDITOR-08):
   - `setBgmVolume(vol)` → `page.value.bgm.volume = vol` — continuous (slider drag), NO pushState (Pitfall 4).
