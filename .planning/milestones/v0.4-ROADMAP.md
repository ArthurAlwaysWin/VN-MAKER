# Milestone v0.4: Voice &amp; Global Font Settings

**Status:** SHIPPED 2026-04-03
**Phases:** 15-18
**Total Plans:** 6

## Overview

Complete voice system (engine + editor + backlog + auto-mode) and global font customization for the Galgame Maker. Players can hear voiced dialogue, replay in backlog, and auto-mode waits for voice. Creators can customize dialogue typography globally.

## Phases

### Phase 15: Voice Engine Foundation

**Goal**: Engine can play voice audio alongside dialogue with independent volume control
**Depends on**: Phase 14 (v0.3 complete)
**Requirements**: VOICE-01, VOICE-04, VOICE-05, VOICE-06
**Plans**: 1 plan

Plans:
- [x] 15-01: Voice audio channel, data model, engine wiring, volume settings

**Success Criteria:**
1. Page JSON stores a `voice` file path per dialogue entry
2. Engine plays the bound voice audio when displaying a dialogue line
3. Engine stops the previous voice when advancing to the next line
4. Player can adjust voice volume independently via the settings page slider
5. Voice volume respects master volume (multiplicative scaling)

### Phase 16: Voice Editor Integration

**Goal**: Creators can bind, preview, and batch-assign voice files to dialogue in the editor
**Depends on**: Phase 15
**Requirements**: VOICE-02, VOICE-03, VOICE-07
**Plans**: 2 plans

Plans:
- [x] 16-01: Voice picker, preview playback, and voice badge in PageInspector
- [x] 16-02: Batch voice matching composable, preview modal, SceneTree buttons

**Success Criteria:**
1. Each dialogue entry in the Inspector shows a voice picker to select or clear a voice file
2. Creator can click play next to a dialogue entry to preview its bound voice
3. Batch naming tool scans audio folder and auto-binds files matching naming convention
4. Batch binding shows matched results for confirmation before applying

### Phase 17: Global Font Settings

**Goal**: Creators can customize dialogue box typography globally with live preview
**Depends on**: None (independent of voice; sequenced after Phase 16 for workflow)
**Requirements**: FONT-01, FONT-02, FONT-03, FONT-04
**Plans**: 2 plans

Plans:
- [x] 17-01: Data model (script store 6-field dialogueBox) + engine consumption
- [x] 17-02: Editor font settings UI + canvas live preview + per-page font override

**Success Criteria:**
1. script.json stores dialogue box font settings with sensible defaults
2. Engine dialogue box renders text using global font settings via CSS custom properties
3. Editor provides font settings UI (font dropdown, size slider, color picker)
4. Changing font settings immediately updates the canvas dialogue box preview

### Phase 18: Voice Polish

**Goal**: Voice playback integrates seamlessly with backlog review and auto-play mode
**Depends on**: Phase 15
**Requirements**: VOICE-08, VOICE-09
**Plans**: 1 plan

Plans:
- [x] 18-01: AudioManager Promise + BacklogScreen replay + auto-mode voice wait + CSS

**Success Criteria:**
1. Backlog screen shows replay button on entries with bound voice; clicking replays
2. Auto-mode waits for voice audio to finish before advancing to the next line
3. Auto-mode advances at normal timing interval when dialogue has no voice bound

## Milestone Summary

**Key Decisions:**
- D-01 (Ph15): Voice stops only when NEW voice starts, not on unvoiced dialogue
- D-02 (Ph15): Voice file stored as pure path string (no object wrapper)
- D-04 (Ph16): Reuse AudioPicker component with mode prop for voice selection
- D-05 (Ph16): Editor voice preview via new Audio() independent of iframe engine
- D-01 (Ph18): Backlog click replaces playing voice (no manual stop needed)
- D-04 (Ph18): Auto wait = max(voiceDuration, textComplete + autoSpeed)
- D-06 (Ph18): playVoice returns Promise for unified completion notification
