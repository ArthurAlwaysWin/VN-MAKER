# Phase 20 — UI Design Contract: Quick Action Bar

> Visual and interaction contract for the QuickActionBar game runtime component. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (pure CSS, vanilla JS DOM) |
| Preset | not applicable |
| Component library | none — game runtime uses plain DOM manipulation |
| Icon library | Lucide (inline SVG, no npm package — D-08) |
| Font | `'Noto Sans SC', 'Segoe UI', 'Microsoft YaHei', sans-serif` (existing) |

## Spacing Scale

Declared values (multiples of 4, scoped to QuickActionBar and toast):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Bar vertical padding top (`padding: 4px 0 8px`) |
| sm | 8px | Gap between icon buttons; bar vertical padding bottom |
| md | 16px | — (not used in this phase) |
| lg | 24px | Toast horizontal padding (`padding: 10px 24px`) |
| xl | 32px | — (not used in this phase) |
| 2xl | 36px | Icon button touch target (width × height) |
| 3xl | 60px | Dialogue box side padding (existing `padding: 0 60px 30px 60px`) |

Exceptions:
- Button size 36×36px (not a multiple of 8, but standard touch target matching the 20px icon + 8px padding each side)
- Icon SVG 20×20px within 24×24 viewBox (Lucide standard)

## Interaction Contract

### Click Behavior (BAR-05)

| Interaction | Behavior |
|-------------|----------|
| Click any bar button | `e.stopPropagation()` — prevents dialogue advance |
| Click `auto` | Toggle auto mode; main.js calls `bar.setAutoActive(bool)` |
| Click `skip` | Toggle skip mode; main.js calls `bar.setSkipActive(bool)` |
| Click `backlog` | Open BacklogScreen with history |
| Click `save` | Stop auto/skip → capture screenshot → open SaveLoadScreen('save') |
| Click `load` | Stop auto/skip → open SaveLoadScreen('load') |
| Click `quicksave` | Capture screenshot → save to quicksave slot → toast "快速存档完成" → enable quickload |
| Click `quickload` | Guard: only if enabled → load quicksave → restore state → toast "快速读档完成" |
| Click `settings` | Stop auto/skip → open SettingsScreen |
| Click disabled button | No action (`pointer-events: none` on `.disabled` class) |

### Keyboard Shortcuts (D-21, D-22)

| Key | Action | Guard |
|-----|--------|-------|
| F5 | Quicksave (same as clicking 快存) | `isPlaying === true` |
| F9 | Quickload (same as clicking 快読) | `isPlaying === true` AND `quickBar.isQuickLoadEnabled` |
| A | Toggle auto (existing) | `isPlaying === true` |
| S | Toggle skip (existing) | `isPlaying === true` |
| L | Toggle backlog (existing) | `isPlaying === true` |
| ESC | Close overlays / toggle dialogue (existing) | Always active |

### Visibility Sync (BAR-04)

| Trigger | Bar Behavior | Mechanism |
|---------|-------------|-----------|
| Dialogue box shows | Bar shows | DOM child — automatic |
| Dialogue box hides | Bar hides | DOM child — automatic |
| Choice page appears | Bar hides | Choice handler hides dialogue box |
| GameMenu opens | Bar remains hidden | Dialogue already hidden |
| SaveLoadScreen opens | Bar remains hidden | Dialogue already hidden |
| SettingsScreen opens | Bar remains hidden | Dialogue already hidden |
| BacklogScreen opens | Bar remains hidden | Dialogue already hidden |
| ESC toggles dialogue | Bar follows | DOM child — automatic |
| Right-click toggles dialogue | Bar follows | DOM child — automatic |
| Screenshot capture | Bar hidden | Dialogue hidden during capture |

### State Management (D-18, D-19, D-20)

| Method | Called by | Effect |
|--------|----------|--------|
| `bar.setAutoActive(true)` | main.js `toggleAuto()` | Add `.active` to auto button |
| `bar.setAutoActive(false)` | main.js `stopAuto()` | Remove `.active` from auto button |
| `bar.setSkipActive(true)` | main.js `toggleSkip()` | Add `.active` to skip button |
| `bar.setSkipActive(false)` | main.js `stopSkip()` | Remove `.active` from skip button |
| `bar.setQuickLoadEnabled(true)` | main.js after quicksave success | Remove `.disabled` from quickload button |
| `bar.setQuickLoadEnabled(false)` | main.js on init (no quicksave) | Add `.disabled` to quickload button |
| `bar.isQuickLoadEnabled` (getter) | main.js F9 handler | Returns boolean for keyboard guard |

### Empty State
- **QuickLoad disabled:** No copy — visual only (`.disabled` class reduces opacity). Tooltip "快读" still visible on hover but button is non-interactive.

### Error State
- **QuickSave failure:** Toast `快速存档失败：{error}` — error message from IPC result appended after colon.
- **QuickLoad on empty:** Prevented by `isQuickLoadEnabled` guard — user never sees an error.

### Destructive Actions
- None in this phase. QuickSave overwrites silently (D-11: single slot, always overwrite, no confirmation).
