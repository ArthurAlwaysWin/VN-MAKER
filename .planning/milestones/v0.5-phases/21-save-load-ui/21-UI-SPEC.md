# Phase 21 — UI Design Contract: Save/Load UI

> Visual and interaction contract for the full-screen save/load interface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (pure CSS, vanilla JS DOM) |
| Preset | not applicable |
| Component library | none — game runtime uses plain DOM manipulation |
| Icon library | none — only Unicode ✕ for delete button (existing pattern) |
| Font | `'Noto Sans SC', 'Segoe UI', 'Microsoft YaHei', sans-serif` (body); `'Noto Serif SC', serif` (title) |

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Page tab gap; inline button padding |
| sm | 8px | — |
| md | 12px | Card inner padding; card content gap; confirm button gap; pagination vertical padding |
| lg | 16px | Grid gap between cards |
| xl | 20px | Header vertical padding |
| 2xl | 24px | Grid vertical padding (top/bottom) |
| 3xl | 40px | Header and grid horizontal padding |

Exceptions:
- Delete button top offset `6px` and right offset `8px` (existing, fine-tuned to card corner).
- Page tab size `32×32px` (touch target for pagination, not a spacing token).

## Interaction Contract

### show(mode, source) Signature (D-10)

| Parameter | Type | Values | Purpose |
|-----------|------|--------|---------|
| mode | `'save' \| 'load'` | Required | Determines save vs load behavior |
| source | `'bar' \| 'menu' \| 'title'` | Required | Determines close routing (D-11) |

### Slot Click Behavior

| Mode | Slot State | Action |
|------|-----------|--------|
| save | occupied | Show inline confirmation "确定覆盖?" (D-07, D-18) |
| save | empty | Call `onSave(slotNum)` directly, then re-render grid (D-18) |
| load | occupied | Call `onLoad(slotNum)`, then `hide()` (D-19) |
| load | empty | No action — card disabled via `pointer-events: none` (D-19) |

### Overwrite Confirmation Flow (D-07)

1. User clicks occupied slot in save mode
2. `.save-confirm-overlay` appended inside the clicked `.save-slot` (absolute positioned)
3. Overlay shows "确定覆盖?" + `[覆盖]` + `[取消]`
4. **覆盖 clicked:** call `onSave(slotNum)`, remove overlay, re-render grid
5. **取消 clicked:** remove overlay, card returns to normal
6. Clicking outside the confirming card: no effect (confirmation stays)

### Delete Confirmation Flow (D-08)

1. User clicks `✕` delete button on occupied slot (any mode)
2. `e.stopPropagation()` on delete button click
3. `.save-confirm-overlay` appended inside the slot card
4. Overlay shows "确定删除?" + `[删除]` (red) + `[取消]`
5. **删除 clicked:** call `onDelete(slotNum)`, remove overlay, re-render grid
6. **取消 clicked:** remove overlay, card returns to normal

### Pagination (D-04, D-05, D-06)

| Interaction | Behavior |
|-------------|----------|
| Click page tab | Set `currentPage`, re-render grid with slots for that page. No animation (D-06) |
| Left arrow key ← | Go to previous page (clamp to page 1) |
| Right arrow key → | Go to next page (clamp to page 12) |
| Page calculation | Page N shows slots `(N-1)*9 + 1` through `N*9` |

### Close Behavior (D-11, D-12)

| Trigger | Behavior |
|---------|----------|
| Click "返回" button | Call `hide()` |
| Press ESC | Call `hide()` (via main.js ESC chain, already highest priority) |
| `hide()` internally | Remove visible class → call `onClose(this._source)` |

### Close Routing by Source (D-11)

| Source | onClose Behavior (in main.js) |
|--------|-------------------------------|
| `'bar'` | No action — return to gameplay |
| `'menu'` | Re-open `gameMenu.show()` |
| `'title'` | No action — title screen still visible underneath |

### Post-Load Behavior

| Source | After Successful Load |
|--------|----------------------|
| Any | `onLoad(slot)` callback in main.js hides title, restores state, sets `isPlaying = true` (existing logic unchanged) |

### No Mode Switching (D-13)

Entry point determines mode. There are no tab switches between save/load within the interface. The save-load screen always shows one mode at a time. This intentionally overrides SLUI-03 per user decision D-13.

### Keyboard Shortcuts (within save-load screen)

| Key | Action | Guard |
|-----|--------|-------|
| ESC | Close save-load screen (source-routed) | Screen is visible |
| ← | Previous page | Screen is visible, `currentPage > 1` |
| → | Next page | Screen is visible, `currentPage < 12` |

Note: These arrow key listeners are scoped to when the save-load screen is visible. They should NOT interfere with other keyboard handlers. Attach on `show()`, detach on `hide()`.

### State Properties

| Property | Type | Purpose |
|----------|------|---------|
| `this.mode` | `'save' \| 'load'` | Current mode |
| `this._source` | `'bar' \| 'menu' \| 'title'` | Close routing context |
| `this._currentPage` | `number` (1-12) | Current pagination page |

### Callback Interface

| Callback | Signature | When Called |
|----------|-----------|------------|
| `onSave` | `(slot: number) => Promise<void>` | User confirms save to slot |
| `onLoad` | `(slot: number) => void` | User clicks occupied slot in load mode |
| `onDelete` | `(slot: number) => Promise<void>` | User confirms delete |
| `onClose` | `(source: string) => void` | Screen hides (for main.js routing) |

### ESC Priority Chain (unchanged position)

```
SaveLoadScreen (z:200) → SettingsScreen → BacklogScreen → GameMenu → Game
```

SaveLoadScreen already occupies the highest overlay priority. No ESC chain changes needed. The existing `saveLoadScreen.hide()` call in the ESC handler will trigger `onClose` routing automatically.

### Visibility Sync

| Trigger | Screen Behavior |
|---------|----------------|
| `show('save', 'bar')` | Capture screenshot first (in main.js before calling show), then display screen |
| `show('save', 'menu')` | GameMenu already visible, capture screenshot, show screen on top |
| `show('load', 'bar')` | Display screen immediately (no screenshot needed) |
| `show('load', 'menu')` | Display screen on top of GameMenu |
| `show('load', 'title')` | Display screen on top of title screen |

### Empty State

- **No saves exist (all slots empty):** 9 empty cards with dashed borders and "— 空 —" text. No special "no saves" banner — the empty grid pattern IS the empty state.
- **Load mode with all empty slots:** Same 9 empty cards but at `opacity: 0.3` with no interaction.

### Error State

- **Save failure:** Toast `存档失败：{error}` — error from IPC. Screen stays open, grid re-rendered.
- **Load failure:** Toast `读档失败：{error}` — screen stays open.
- **Delete failure:** Toast `删除失败：{error}` — screen stays open, confirmation overlay removed.
- **Thumbnail load failure:** `<img>` falls back to `rgba(0,0,0,0.3)` background (existing img bg).

### Destructive Actions
