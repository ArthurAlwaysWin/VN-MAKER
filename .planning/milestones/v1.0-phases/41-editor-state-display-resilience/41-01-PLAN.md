**Add helper function** `findInheritedExpression(charId)` before `getCharImage()`:

```javascript
/**
 * Walk backward through previous pages in current scene to find
 * the last explicit expression for a character (editor-side inheritance).
 * @param {string} charId
 * @returns {string|null} expression name or null
 */
function findInheritedExpression(charId) {
  const scene = editor.currentScene.value;
  const pageIdx = editor.selectedPageIndex.value;
  if (!scene?.pages || pageIdx <= 0) return null;

  for (let i = pageIdx - 1; i >= 0; i--) {
    const prevPage = scene.pages[i];
    const prevChar = (prevPage.characters || []).find(c => c.id === charId);
    if (prevChar?.expression) {
      // Validate the inherited expression still exists in character definition
      const charDef = script.data?.characters?.[charId];
      if (charDef?.expressions?.[prevChar.expression]) {
        return prevChar.expression;
      }
    }
  }
  return null;
}
```

**Replace `getCharImage()`** with:

```javascript
function getCharImage(char) {
  const charDef = script.data?.characters?.[char.id];
  if (!charDef?.expressions) return null;
  const expressions = charDef.expressions;

  // Resolution chain: explicit → validate exists (D-08) → inherited → first → null
  let resolvedExpr = char.expression;

  // D-08: Validate explicit expression still exists
  if (resolvedExpr && !expressions[resolvedExpr]) {
    resolvedExpr = null;
  }

  // Inherit from previous pages in scene (UI-03)
  if (!resolvedExpr) {
    resolvedExpr = findInheritedExpression(char.id);
  }

  // Fallback to first expression
  if (!resolvedExpr) {
    resolvedExpr = Object.keys(expressions)[0] || null;
  }

  if (!resolvedExpr || !expressions[resolvedExpr]) return null;
  return resolveAsset(expressions[resolvedExpr]);
}
```

This ensures: (1) stale expression names don't break the canvas — they fallback gracefully (D-08/UI-04), (2) characters with no explicit expression on current page show the inherited expression from previous pages (UI-03), (3) characters with no inherited expression show the first available expression, (4) characters with no expressions at all show the placeholder 🧑 (existing template `v-else` handles this).

Replace the current `addPage()` function body:

```javascript
function addPage(sceneId, afterIndex = -1) {
  const scene = data.value?.scenes?.[sceneId];
  if (!scene) return null;
  const newPage = createDefaultPage();
  const insertAt = afterIndex >= 0 ? afterIndex + 1 : scene.pages.length;

  // D-03: PPT-style copy from previous page in same scene
  const prevPage = scene.pages[insertAt - 1];
  if (prevPage) {
    newPage.characters = JSON.parse(JSON.stringify(prevPage.characters || []));
    newPage.background = prevPage.background || null;
    newPage.bgm = prevPage.bgm ? JSON.parse(JSON.stringify(prevPage.bgm)) : null;
    // D-03: dialogues and voice intentionally left as default empty
    // D-03: se intentionally left as null
  }
  // D-04: If no prevPage (e.g., insertAt=0), newPage stays blank from createDefaultPage()

  scene.pages.splice(insertAt, 0, newPage);
  pushState();
  return { page: newPage, index: insertAt };
}
```

Key details:
- `JSON.parse(JSON.stringify())` for deep copy per project convention (not structuredClone)
- Characters array deep-copied to avoid shared references (positions/scales are independent per page)
- Background is a string path (shallow copy sufficient) but guarded with `|| null`
- BGM is an object `{ file, volume }` — deep-copied to avoid shared reference
- New page keeps its own fresh `id`, `name`, `transition`, empty `dialogues`, null `se` from `createDefaultPage()` — these are NOT copied (Pitfall 3)
- D-04 satisfied naturally: `addScene()` calls `createDefaultPage()` directly (not `addPage()`), and when `addPage()` is first called in a 1-page scene, `prevPage = scene.pages[0]` which is the blank first page — copying a blank page yields another blank-ish page (with empty characters array)
