# Plan 44-01: SettingsScreen Structured Mode

## Goal
Add `_renderStructured()` to SettingsScreen that auto-renders SETTING_DEFS grouped into tabs
when `elements[]` is empty but `header`/`tabBar`/`contentArea` config exists.
Structured mode uses widgetStyles (when set) for Tab/Toggle/Slider rendering.

## Context
- SettingsScreen.js: ~512 lines, has _renderCustom + _renderDefault
- Phase 42 already added: widgetStyles branching in _buildSlider/_buildToggle, createTabBar import
- SETTING_DEFS: 9 settings (6 sliders: masterVolume/bgmVolume/seVolume/voiceVolume/textSpeed/autoSpeed/dialogueOpacity, 2 selects: windowMode/skipMode, 0 toggles currently)
- Design spec Section 5.4: header/tabBar/contentArea/footer schema

## Setting Groups Constant
```js
const SETTING_GROUPS = {
  '声音': ['master-volume', 'bgm-volume', 'se-volume', 'voice-volume'],
  '画面': ['dialogue-opacity', 'window-mode'],
  '游戏': ['text-speed', 'auto-speed', 'skip-mode'],
};
```
Note: Group keys match tabBar.tabs default labels. When config.tabBar.tabs has custom labels,
the groups map by index (tabs[0] → first group, tabs[1] → second, etc.).

## Tasks

### Task 1: Add setting groups constant and structured mode routing
**File:** `src/ui/SettingsScreen.js`
**Action:**
1. Add `SETTING_GROUPS` constant (array-of-arrays to preserve order):
```js
const SETTING_GROUP_KEYS = [
  ['master-volume', 'bgm-volume', 'se-volume', 'voice-volume'],
  ['dialogue-opacity', 'window-mode'],
  ['text-speed', 'auto-speed', 'skip-mode'],
];
const DEFAULT_TAB_LABELS = ['声音', '画面', '游戏'];
```
2. Add `this._activeTab = 0;` in constructor
3. Modify show() routing:
```js
show() {
  // slider CSS injection (existing)
  if (this.customLayout?.elements?.length > 0) {
    this._renderCustom(this.customLayout);
  } else if (this.customLayout?.header || this.customLayout?.tabBar || this.customLayout?.contentArea) {
    this._renderStructured(this.customLayout);
  } else {
    this._renderDefault();
  }
  // show animation (existing)
}
```

### Task 2: Implement _renderStructured() — header
**File:** `src/ui/SettingsScreen.js`
**Action:** Add `_renderStructured(layout)` method:
1. Clear el, reset styles
2. Render header section:
   - Container div with class `settings-structured-header`
   - If layout.header.backgroundImage → background-image via resolvePath
   - Height from layout.header.height (default 90)
   - Title: text, fontSize, fontFamily, color, x/y positioning
3. Add close button in header (top-right)

### Task 3: Implement _renderStructured() — tab bar
**File:** `src/ui/SettingsScreen.js`
**Action:**
1. Import createTabBar if not already imported (it is from Phase 42)
2. Get tab labels from layout.tabBar.tabs or DEFAULT_TAB_LABELS
3. If _widgetStyles → use createTabBar(labels, _widgetStyles.tab, onSelect)
4. If no _widgetStyles → render fallback tab buttons (simple button row)
5. Tab bar positioned at layout.tabBar.y, height layout.tabBar.height, background layout.tabBar.background
6. onSelect callback: `(index) => { this._activeTab = index; this._renderStructuredContent(layout); }`

### Task 4: Implement _renderStructured() — content area
**File:** `src/ui/SettingsScreen.js`
**Action:** Add `_renderStructuredContent(layout)` method:
1. Get or create content container div with class `settings-structured-content`
2. Position: layout.contentArea.x/y/width/height, overflow-y: auto
3. Get active group keys: `SETTING_GROUP_KEYS[this._activeTab]`
4. For each setting key in the active group:
   - Get def from SETTING_DEFS
   - Create wrapper div with setting label
   - Call appropriate builder based on def.type:
     - 'slider' → _buildSlider(wrapper, def, cfg, DEFAULT_SETTING_STYLE)
     - 'toggle' → _buildToggle(wrapper, def, cfg, DEFAULT_SETTING_STYLE)
     - 'select' → _buildSelect(wrapper, def, cfg, DEFAULT_SETTING_STYLE)
5. Clear and re-populate on each call (tab switch re-renders content only)

### Task 5: Implement _renderStructured() — footer
**File:** `src/ui/SettingsScreen.js`
**Action:**
1. If layout.footer exists:
   - Create footer container div
   - Height from layout.footer.height
   - For each button in layout.footer.buttons:
     - Create button element at x/y position with text
     - If id contains 'title' → bind to this.onTitle callback (if exists)
     - If id contains 'close' or 'back' → bind to this.hide()
