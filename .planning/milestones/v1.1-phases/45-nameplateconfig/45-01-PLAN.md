# Plan 45-01: DialogueBox Nameplate Styles

## Goal
Add `setNameplateStyle(style)` to DialogueBox supporting 3 visual styles for the speaker
name display: inline (default/current), floating (bubble outside box), banner (full-width bar).

## Context
- DialogueBox.js: ~230 lines. DOM structure:
  ```html
  <div id="dialogue-box">
    <div class="dialogue-name-plate">
      <span class="dialogue-speaker-name"></span>
    </div>
    <div class="dialogue-text-area">
      <span class="dialogue-text"></span>
      <span class="dialogue-indicator">▼</span>
    </div>
  </div>
  ```
- show(data) renders speakerName into nameEl, toggles name-plate visibility
- _applyFontSettings handles nameplateFontSize/nameplateFontFamily/nameplateColor
- sanitize.js available for CSS sanitization

## Design Spec (Section 5.5)
```json
{
  "nameplateStyle": "inline",
  "showSpeakerPortrait": false,
  "portraitSize": 48,
  "portraitRadius": 24
}
```
- "inline" — name above text inside box (current default)
- "floating" — floating bubble at top-left outside dialogue box
- "banner" — full-width banner bar (千恋万花 style)

Note: showSpeakerPortrait is out of scope for v1.1 (see REQUIREMENTS.md Out of Scope).

## Tasks

### Task 1: Add setNameplateStyle() method and CSS injection
**File:** `src/ui/DialogueBox.js`
**Action:**
1. Add `this._nameplateStyle = 'inline';` in constructor
2. Add `this._nameplateCssInjected = false;` in constructor
3. Add method:
```js
setNameplateStyle(style) {
  const valid = ['inline', 'floating', 'banner'];
  this._nameplateStyle = valid.includes(style) ? style : 'inline';
  if (!this._nameplateCssInjected && this._nameplateStyle !== 'inline') {
    this._injectNameplateCSS();
    this._nameplateCssInjected = true;
  }
}
```
4. Add `_injectNameplateCSS()` method that creates a <style> element with:

```css
/* Floating style — bubble positioned above dialogue box */
#dialogue-box .dialogue-name-plate.nameplate-floating {
  position: absolute;
  top: -32px;
  left: 16px;
  background: rgba(0, 0, 0, 0.75);
  padding: 4px 14px;
  border-radius: 12px;
  z-index: 1;
  display: inline-block;
  width: auto;
}

/* Banner style — full-width bar across top of dialogue box */
#dialogue-box .dialogue-name-plate.nameplate-banner {
  width: 100%;
  background: rgba(0, 0, 0, 0.6);
  padding: 6px 20px;
  border-radius: 0;
  box-sizing: border-box;
}
```

### Task 2: Apply nameplate style class in show()
**File:** `src/ui/DialogueBox.js`
**Action:** In show(), where speakerName is handled (around line 69-81):
- Before showing name-plate, remove all nameplate-* classes
- Add class based on _nameplateStyle: `nameplate-${this._nameplateStyle}`
- The 'inline' class adds no extra styling (matches current behavior)

```js
// Apply nameplate style class
const namePlate = this.nameEl.parentElement;
namePlate.classList.remove('nameplate-inline', 'nameplate-floating', 'nameplate-banner');
namePlate.classList.add(`nameplate-${this._nameplateStyle}`);
```

### Task 3: Unit tests
**File:** `tests/dialogueBoxNameplate.test.js`
**Action:** Create tests:
1. Default _nameplateStyle is 'inline'
2. setNameplateStyle('floating') sets _nameplateStyle to 'floating'
3. setNameplateStyle('banner') sets _nameplateStyle to 'banner'
4. setNameplateStyle('invalid') falls back to 'inline'
5. show() with inline style → name-plate has class 'nameplate-inline'
6. show() with floating style → name-plate has class 'nameplate-floating'
7. show() with banner style → name-plate has class 'nameplate-banner'
8. show() without speakerName → name-plate hidden regardless of style
9. CSS is injected when non-inline style is set
10. NAMEPLATE-01: inline style produces same behavior as no setNameplateStyle call

## NAMEPLATE-01 Verification
When _nameplateStyle is 'inline', the name-plate div gets class 'nameplate-inline' which
has NO CSS rules → identical to current behavior. This is the backward-compatible default.
