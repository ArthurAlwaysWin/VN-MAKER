# Milestone v1.4: 演出力升级

**Status:** ✅ SHIPPED 2026-04-22
**Phases:** 61-70
**Total Plans:** 20

## Overview

受限范围的演出升级：在不引入新动画或渲染依赖的前提下，为编辑器和运行时补齐 **预设角色动画、单页镜头效果、扩展背景转场、runtime-backed iframe 真预览**，并通过 docs-only verification backfill 关闭 v1.4 的全部审计 orphan gap。

## Milestone Summary

**Key Decisions:**

- 采用 **preset-based cinematic upgrade**，而不是自由动画平台，控制 v1.4 范围与可靠性。
- 采用 **runtime-backed iframe preview** 作为单一事实来源，拒绝 editor-only 伪预览。
- 采用 **stage-layer-only camera scope**，保证镜头效果不影响对话和 UI 可读性。
- 采用 **focused PREV-05 regression gate**，避免把与 v1.4 无关的 repo-wide Vitest debt 混入 milestone closure。
- 采用 **docs-only verification backfill phases (68-70)** 修复审计证据链，而不是重开实现 scope。

**Issues Resolved:**

- 角色动画、镜头、转场、预览和 editor controls 的实现链在 61-67 闭环落地。
- 18 个 requirement 的 orphaned audit gap 已全部关闭。
- `PREV-05` 现已同时具备 67-level focused proof 与 70-level final owner proof。

**Issues Deferred:**

- `usePageEditor.js` 与 `PageInspector.vue` 的 preview preflight 规则仍有重复实现。
- repo-wide `npx vitest run` 仍包含与 v1.4 无关的 legacy test collection / `tests/mainConfigRouting.test.js` 债务。

**Technical Debt Incurred:**

- Phase 68-70 引入 docs-only closeout pattern；后续若复用同类 backfill，必须继续保持“final requirement owner 必须有自己的 verification anchor”。
