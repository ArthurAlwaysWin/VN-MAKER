# Requirements: Galgame Maker v1.6 完整主题包与图片主题产出

**Defined:** 2026-04-27
**Core Value:** 开发者不碰逻辑 — 只做视觉设计，引擎处理一切游戏逻辑

## v1 Requirements

本里程碑聚焦把 v1.5 已打通的图片 UI 能力升级成 **可分发、可复用、可一键套用** 的完整主题包系统。  
完成边界不是“多几个图片槽位”，而是：**完整主题包契约 + 安装 / 应用 / 导出链路 + 5 套可开箱即用的完整图片主题**。

### Theme Package Contract

- [x] **PKG-01**: 用户可以导入包含主题配置与全部 UI 图片资产的 `.gmtheme` 完整主题包，所需文件会自动安装到当前项目中，不需要手动补路径
- [x] **PKG-02**: 用户可以将当前项目主题导出为自包含的 `.gmtheme` 包，包内会携带主题实际引用的全部 UI 图片资产
- [x] **PKG-03**: 用户导入损坏、不兼容或引用非法路径的主题包时，编辑器会在写入项目之前明确阻止并说明原因
- [x] **PKG-04**: 用户重复导入同一主题包时，主题资产会在自己的命名空间内稳定更新，不会不断生成 `-1/-2/-3` 之类的重复文件
- [x] **PKG-05**: 用户仍可以导入旧 `.theme` 包，但编辑器会明确标记它是“兼容导入 / 部分主题”，并展示缺失的覆盖范围

### Theme Apply & Project Integration

- [x] **APP-01**: 用户应用内置主题或导入主题时，两者都走同一条安装 / 应用路径，并在项目内落成可导出、可搬运的本地主题资产
- [x] **APP-02**: 用户应用完整主题时，主题负责的 UI 区块会原子性整体替换，并支持一次撤销恢复到应用前状态
- [x] **APP-03**: 用户应用完整主题、保存项目并重新打开后，主题在编辑器预览、运行时试玩和导出成品中的结果保持一致，不会出现资源丢失或局部失配

### Theme Browser & Selection UX

- [x] **BRW-01**: 用户在主题浏览器里能看到每个主题的卡片预览图、名称、作者、版本、来源（内置 / 导入）、兼容信息和完整/部分主题标识
- [x] **BRW-02**: 用户在应用主题前可以查看它覆盖哪些 UI 面，及本次操作会覆盖当前项目中的哪些范围，而不需要先实际应用才知道结果
- [x] **BRW-03**: 用户可以清楚分辨一个主题当前处于“内置可用 / 已导入 / 当前已应用 / 仅兼容部分主题”中的哪种状态

### Shipped Complete Themes

- [x] **THM-01**: 产品先交付 1 套 golden theme，完整覆盖 v1.6 定义的主题覆盖面，并用它验证导入、应用、保存、重开和导出的整链路
- [x] **THM-02**: 产品最终交付 5 套完整图片主题，这 5 套主题都达到同一覆盖标准，而不是只有 1 套完整、其余只是换色或半成品
- [x] **THM-03**: 这 5 套已交付主题在视觉识别上有明确差异，至少体现在材质语言、轮廓语言和整体风格倾向上，而不只是配色不同

## v2 Requirements

延后到后续 milestone，不进入本次 roadmap。

### Theme Scope Expansion

- **TSCP-01**: 用户可以为单个 scene 或 page 覆盖项目级完整主题
- **TSCP-02**: 用户可以在主题包中携带字体资源并自动处理字体兼容 / 许可提示

### Theme Ecosystem

- **TECO-01**: 用户可以浏览和导入在线主题市场中的社区主题
- **TECO-02**: 用户可以跨设备同步自己安装过的主题库

### Advanced Preview

- **TPRV-01**: 用户在未应用主题前，可以通过真实 iframe live preview 查看该主题的运行时效果

## Out of Scope

明确排除，避免 v1.6 范围膨胀。

| Feature | Reason |
|---------|--------|
| 在线主题市场 / 云同步 | 先把本地主题包契约与安装链路跑稳，再考虑分发生态 |
| 每场景 / 每页面局部主题覆盖 | 会把“完整主题包”拆成更复杂的覆盖系统，超出本 milestone 核心目标 |
| 主题包内嵌字体资源 | 体积、许可和跨平台兼容复杂度高，延后处理 |
| 未应用主题的真实 iframe live preview | 当前 milestone 先用卡片预览图 + 覆盖范围说明控制复杂度 |
| 把主题系统扩展成自由画布式 UI 设计器 | v1.6 目标是完整主题包系统，不是新 UI 编辑器平台 |
