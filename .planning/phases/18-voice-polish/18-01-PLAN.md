## Task 1: AudioManager — playVoice returns Promise (D-06)

**File:** `src/engine/AudioManager.js`

Refactor `playVoice()` (lines 121-127) to return a Promise that resolves when the voice finishes playing.

**Replace lines 121-127 with:**

```javascript
playVoice(file) {
  this.stopVoice();
  if (!file) return Promise.resolve();
  this._voice = new Audio(this.basePath + file);
  this._voice.volume = this.voiceVolume;

  return new Promise((resolve) => {
    const voice = this._voice;
    voice.onended = () => resolve();
    voice.onerror = () => resolve();
    voice.play().catch(() => resolve());
  });
}
```

**Key behaviors:**
- Returns `Promise.resolve()` immediately for falsy file (D-06 edge case)
- Wraps onended as the primary resolve trigger
- onerror and play().catch() also resolve (never reject — callers don't need error handling)
- stopVoice() sets `this._voice = null`, which means the old element's onended still fires but that's fine — it just resolves the old promise

**Also update `stopVoice()`** (lines 132-138) to trigger resolve of pending promise:

```javascript
stopVoice() {
  if (this._voice) {
    const voice = this._voice;
    this._voice = null;
    voice.pause();
    voice.currentTime = 0;
    // Trigger onended to resolve any pending promise
    if (voice.onended) voice.onended();
  }
}
```

**Key:** Setting `this._voice = null` before triggering onended prevents re-entrant stopVoice calls.

**Verification (Task 1):**
- `playVoice('test.wav')` returns a Promise
- `playVoice(null)` returns a resolved Promise (synchronous)
- `playVoice('')` returns a resolved Promise
- `stopVoice()` after `playVoice()` resolves the promise
- `playVoice()` called twice — first promise resolves (via stopVoice inside playVoice)

```javascript
_playVoice(entryEl, btn, voiceFile) {
  // If clicking the same entry that's playing → stop (D-02 ■ click)
  if (this._playingEntry === entryEl) {
    this._stopCurrentVoice();
    return;
  }

  // Stop previous if any (D-01 — clicking another replaces)
  this._stopCurrentVoice();

  // Start new voice
  this._playingEntry = entryEl;
  btn.textContent = '■';
  entryEl.classList.add('backlog-playing');

  this.audio.playVoice(voiceFile).then(() => {
    // Only restore if this entry is still the playing one
    if (this._playingEntry === entryEl) {
      this._restoreEntry(entryEl, btn);
    }
  });
}

_stopCurrentVoice() {
  if (this._playingEntry) {
    const entry = this._playingEntry;
    const btn = entry.querySelector('.backlog-voice-btn');
    this._playingEntry = null;
    this.audio.stopVoice();
    if (btn) btn.textContent = '▶';
    entry.classList.remove('backlog-playing');
  }
}

_restoreEntry(entryEl, btn) {
  this._playingEntry = null;
  btn.textContent = '▶';
  entryEl.classList.remove('backlog-playing');
}
```

### 4c. Rewrite startAutoTimer for voice awareness

Replace `startAutoTimer()` (lines 338-352) with a voice-aware implementation:

```javascript
function startAutoTimer() {
  clearAutoTimer();

  // Text completion wait (existing behavior)
  const textWait = new Promise((resolve) => {
    const check = setInterval(() => {
      if (dialogueBox.isComplete()) {
        clearInterval(check);
        // Then wait autoSpeed
        const t = setTimeout(resolve, config.get('autoSpeed'));
        autoTimer = t;
      }
    }, 100);
    autoTimer = check;
  });

  // Voice wait: if voice is playing, wait for it + micro-delay (D-04, D-05)
  const voiceWait = currentVoicePromise
    ? currentVoicePromise.then(() => new Promise(r => setTimeout(r, VOICE_END_DELAY)))
    : null;

  // D-04: max(voiceDuration, textComplete + autoSpeed)
  const waits = voiceWait ? [textWait, voiceWait] : [textWait];
  Promise.all(waits).then(() => {
    if (autoMode && engine.waiting) {
      engine.next();
    }
  });
}
```

**Logic explanation (D-04):**
- `textWait`: polls isComplete(), then waits autoSpeed ms. This is the existing timer behavior.
- `voiceWait`: waits for playVoice Promise + 300ms. Only created when voice exists.
- `Promise.all([textWait, voiceWait])`: both must finish = whoever is longer wins = `max()` semantics.
- When no voice: only `[textWait]` → normal behavior unchanged (VOICE-09).

**Note on clearAutoTimer():** The existing `clearAutoTimer()` (lines 354-360) clears interval/timeout via `autoTimer`. The new implementation still stores the active timer in `autoTimer`, so clearing works. However the Promise.all won't be cancelled — but after clearAutoTimer, `autoMode` will be false, so the `if (autoMode && engine.waiting)` guard prevents advancement. This is correct behavior.
