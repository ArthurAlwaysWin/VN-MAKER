**3. Export `hexToHsl(hex)` — standard cylindrical-coordinate conversion:**
- Input: `'#rrggbb'` string
- Output: `[h, s, l]` array where h=0-360, s=0-100, l=0-100 (all rounded integers)
- Handle edge case: when delta === 0 (achromatic), return `[0, 0, l]`
- Implementation per research Example 3 (lines 546-562):

  - Compute max, min, delta
  - Lightness = (max + min) / 2
  - Saturation: if l > 0.5 then d/(2-max-min) else d/(max+min)
  - Hue: depends on which channel is max (r→(g-b)/d, g→(b-r)/d+2, b→(r-g)/d+4), divide by 6

**4. Export `hslToHex(h, s, l)` — reverse conversion:**
- Input: h (0-360), s (0-100), l (0-100)
- Output: `'#rrggbb'` lowercase hex string
- Implementation per research Example 3 (lines 564-573):
  - Normalize: s /= 100; l /= 100
  - a = s * Math.min(l, 1 - l)
  - f(n) = l - a * Math.max(Math.min(k - 3, 9 - k, 1), -1) where k = (n + h/30) % 12
  - toHex(c) = Math.round(c * 255).toString(16).padStart(2, '0')

**4. Internal sRGB linearization — exact WCAG 2.x formula (do NOT hand-roll):**
```javascript
// ─── sRGB Linearization (WCAG 2.x spec) ────────────────

function linearize(channel) {
  const c = channel / 255;
  return c <= 0.04045 ? c / 12.92 : Math.pow((c + 0.055) / 1.055, 2.4);
}

function relativeLuminance(r, g, b) {
  return 0.2126 * linearize(r) + 0.7152 * linearize(g) + 0.0722 * linearize(b);
}
```
These coefficients (0.2126, 0.7152, 0.0722) and the 0.04045 threshold are from the WCAG specification — do NOT modify them.

**5. Export `contrastRatio(hex1, hex2)` (D-10):**
```javascript
/**
 * Calculate WCAG 2.x contrast ratio between two colors.
 *
 * @param {string} hex1 — '#rrggbb'
 * @param {string} hex2 — '#rrggbb'
 * @returns {number} — contrast ratio ≥ 1.0 (e.g., 4.5 means 4.5:1)
 */
export function contrastRatio(hex1, hex2) {
  const L1 = relativeLuminance(...hexToRgb(hex1));
  const L2 = relativeLuminance(...hexToRgb(hex2));
  const [lighter, darker] = L1 > L2 ? [L1, L2] : [L2, L1];
  return (lighter + 0.05) / (darker + 0.05);
}
```

**6. Internal `binarySearchL(h, s, lo, hi, bgHex, target)` function:**
- Binary search for a lightness value between `lo` and `hi` that achieves `>= target` contrast against `bgHex`
- Max 20 iterations, precision epsilon = 0.5 (lightness units)
- When checking if a candidate meets the target and the bg is DARK (luminance < 0.5): we need to search AWAY from bg lightness to increase contrast. For a dark bg, lighter fg increases contrast, so if ratio >= target we try to get closer to original by narrowing toward lo (getting darker/closer to original). For a light bg, darker fg increases contrast.
- Simpler approach: just do pure binary search — if ratio >= target, record result and try to get closer to original. The direction of "closer" depends on which half we move toward:

```javascript
const MAX_ITERATIONS = 20;
const PRECISION = 0.5;

function binarySearchL(h, s, lo, hi, bgHex, target) {
  let resultHex = null;
  let resultL = null;
  let resultRatio = 0;

  for (let i = 0; i < MAX_ITERATIONS; i++) {
    const mid = (lo + hi) / 2;
    const hex = hslToHex(h, s, mid);
    const ratio = contrastRatio(hex, bgHex);

    if (ratio >= target) {
      resultHex = hex;
      resultL = mid;
      resultRatio = ratio;
      // Move closer to the original: narrow toward the "original" side
      // For lighter search (origL→100): lo=origL, hi=100; passing → try lower (hi=mid)
      // For darker search (0→origL): lo=0, hi=origL; passing → try higher (lo=mid)
      // Since caller controls lo/hi, "closer to original" = mid moves toward the start boundary
      hi = mid;
    } else {
      lo = mid;
    }

    if (hi - lo < PRECISION) break;
  }

  if (!resultHex) return null;
  return { hex: resultHex, ratio: resultRatio, l: resultL };
}
```

Wait — the issue is that for the "lighter" search, `lo` is origL and `hi` is 100. If mid passes, we want to get closer to origL (lower), so `hi = mid` is correct. For the "darker" search, `lo` is 0 and `hi` is origL. If mid passes, we want to get closer to origL (higher), so `lo = mid` is correct. But with a single function we need to handle both directions.

**Correct implementation — two internal calls with direction-aware narrowing:**

```javascript
function binarySearchLighter(h, s, origL, bgHex, target) {
  let lo = origL;
  let hi = 100;
  let resultHex = null;
  let resultL = null;
  let resultRatio = 0;

  for (let i = 0; i < MAX_ITERATIONS; i++) {
    const mid = (lo + hi) / 2;
    const hex = hslToHex(h, s, mid);
    const ratio = contrastRatio(hex, bgHex);

    if (ratio >= target) {
      resultHex = hex;
      resultL = mid;
      resultRatio = ratio;
      hi = mid; // try closer to original (lower L)
    } else {
      lo = mid; // need more contrast (higher L)
    }

    if (hi - lo < PRECISION) break;
  }

  if (!resultHex) return null;
  return { hex: resultHex, ratio: resultRatio, l: resultL, direction: 'lighter' };
}

function binarySearchDarker(h, s, origL, bgHex, target) {
  let lo = 0;
  let hi = origL;
  let resultHex = null;
  let resultL = null;
  let resultRatio = 0;

  for (let i = 0; i < MAX_ITERATIONS; i++) {
    const mid = (lo + hi) / 2;
    const hex = hslToHex(h, s, mid);
    const ratio = contrastRatio(hex, bgHex);

    if (ratio >= target) {
      resultHex = hex;
      resultL = mid;
      resultRatio = ratio;
      lo = mid; // try closer to original (higher L)
    } else {
      hi = mid; // need more contrast (lower L)
    }

    if (hi - lo < PRECISION) break;
  }

  if (!resultHex) return null;
  return { hex: resultHex, ratio: resultRatio, l: resultL, direction: 'darker' };
}
```

**7. Export `autoFix(fgHex, bgHex, target)` (D-10, D-11):**
```javascript
/**
 * Auto-fix foreground color to meet target contrast ratio against background.
 * Uses binary search on HSL lightness, trying both lighter and darker directions.
 * Returns the direction closer to the original color (D-11).
 *
 * @param {string} fgHex — foreground '#rrggbb'
 * @param {string} bgHex — background '#rrggbb'
 * @param {number} [target=4.5] — minimum contrast ratio (4.5 for normal text, 3.0 for large text)
 * @returns {{ hex: string, ratio: number, direction: 'none'|'lighter'|'darker' }|null}
 *   null if neither direction can achieve the target (extremely rare)
 */
export function autoFix(fgHex, bgHex, target = 4.5) {
  // Already passes — no fix needed
  const currentRatio = contrastRatio(fgHex, bgHex);
  if (currentRatio >= target) {
    return { hex: fgHex, ratio: currentRatio, direction: 'none' };
  }

  const [h, s, origL] = hexToHsl(fgHex);

  const lighter = binarySearchLighter(h, s, origL, bgHex, target);
  const darker = binarySearchDarker(h, s, origL, bgHex, target);

  if (!lighter && !darker) return null;
  if (!lighter) return darker;
  if (!darker) return lighter;

  // Return the one closer to original lightness
  return Math.abs(lighter.l - origL) <= Math.abs(darker.l - origL)
    ? lighter : darker;
}
```

**Constants:**
- `MAX_ITERATIONS = 20` — sufficient for 0.5 precision across 0-100 range (2^20 >> 200)
- `PRECISION = 0.5` — lightness unit precision (visually imperceptible at this granularity)
