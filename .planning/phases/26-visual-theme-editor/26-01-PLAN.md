Key functions:
- `getMergedTokens()` — returns `{ ...DEFAULT_TOKENS, ...(script.getTheme()?.tokens ?? {}) }` 
- `setToken(key, value)` — mutates `script.getTheme().tokens[key] = value`, calls `sendThemeToPreview()`
- `setTokenBatch(partial)` — `Object.assign(script.getTheme().tokens, partial)`, calls `sendThemeToPreview()`
- `commitTheme()` — deep clones theme via `JSON.parse(JSON.stringify(script.getTheme()))` then calls `script.updateTheme(clone)` to push into undo stack
- `resetTheme()` — calls `script.updateTheme({ tokens: {} })`, calls `sendThemeToPreview()`
- `sendThemeToPreview()` — 200ms debounce (per D-06), sends `{ type: 'update-theme', theme: JSON.parse(JSON.stringify(script.getTheme())) }` to `iframeRef.value.contentWindow.postMessage(msg, '*')`. Guards: skip if `!iframeRef.value?.contentWindow` or `!isEngineReady.value`.
- `flushPreview()` — clears debounce timer and sends immediately (call on commit/undo/redo to avoid stale preview per Pitfall 2)
- `onEngineMessage(event)` — handles `{ type: 'ready' }` messages: sets `isEngineReady.value = true`, then sends `start` message with full script snapshot + first scene, then immediately sends `update-theme` with current theme. Guard: `event.source !== iframeRef.value?.contentWindow` → ignore.
- `startEngine()` — sends `{ type: 'start', script: snapshot, sceneId: firstSceneId, pageIndex: 0, previewMode: true }` to iframe. Uses `JSON.parse(JSON.stringify(script.data))` for snapshot. Gets first scene via `Object.keys(script.data.scenes)[0]`.

```javascript
const editor = useThemeEditor();

const TOKEN_GROUPS = [
  { id: 'core', label: '🎨 核心色', keys: ['primary', 'primary-subtle', 'danger', 'danger-hover', 'accent', 'accent-border', 'shadow', 'title-glow', 'save-title', 'load-title'] },

  { id: 'borders', label: '📏 边框', keys: ['border', 'border-hover', 'border-active'] },
  { id: 'backgrounds', label: '🖼️ 背景', keys: ['dialogue-bg', 'panel-bg', 'menu-bg', 'card-bg', 'card-bg-hover', 'title-bg', 'confirm-bg'] },
  { id: 'buttons', label: '🔘 按钮', keys: ['btn-bg', 'btn-text', 'btn-border', 'btn-hover-bg', 'btn-hover-text', 'btn-hover-border'] },
  { id: 'fonts', label: '🔤 字体', keys: ['font-body', 'font-display'] },
  { id: 'radii', label: '⭕ 圆角', keys: ['radius', 'radius-lg'] },
  { id: 'blur', label: '🌫️ 模糊', keys: ['blur'] },
  { id: 'controls', label: '🎛️ 控件', keys: ['slider-track', 'slider-thumb', 'scrollbar'] },
  { id: 'speaker', label: '💬 说话人', keys: ['speaker-shadow'] },
];

// Token type detection based on DEFAULT_TOKENS values (per D-05)
function getTokenType(key) {
  if (key.startsWith('font-')) return 'font';
  if (key === 'radius' || key === 'radius-lg' || key === 'blur') return 'slider';
  const def = DEFAULT_TOKENS[key];
  if (def.startsWith('linear-gradient')) return 'gradient';
  if (def.startsWith('rgba')) return 'color-alpha';
  return 'color';
}

// Determine if this text token needs a WCAG contrast badge (D-18)
const TEXT_KEYS = new Set(['text', 'text-heading', 'text-secondary', 'text-muted', 'text-dim', 'text-faint']);
function needsContrast(key) { return TEXT_KEYS.has(key); }
```

**3. `ColorTokenRow.vue`** — Color picker + hex input + optional opacity slider (per D-04, D-05).

Props: `defineProps({ tokenKey: String, hasAlpha: Boolean, showContrast: Boolean })`

Script logic:
- Import `useThemeEditor` composable, `ContrastBadge` component.
- `const editor = useThemeEditor();`
- Computed `currentValue`: `editor.getMergedTokens()[props.tokenKey]`
- Helper `parseRgba(rgba)`: parse `rgba(r, g, b, a)` → `{ hex: '#rrggbb', alpha: 0-100 }`. Regex: `/rgba?\(\s*(\d+)\s*,\s*(\d+)\s*,\s*(\d+)\s*(?:,\s*([\d.]+))?\)/`. Convert r/g/b to hex padStart(2, '0'). alpha = `m[4] !== undefined ? Math.round(parseFloat(m[4]) * 100) : 100`.
- Helper `buildRgba(hex, alpha)`: parse hex slices → `rgba(r, g, b, ${(alpha/100).toFixed(2)})`.
- Computed `hexValue`: if `hasAlpha` and `currentValue` starts with 'rgba', return `parseRgba(currentValue).hex`; else return `currentValue` (already hex).
- Computed `alphaValue`: if `hasAlpha`, return `parseRgba(currentValue).alpha`; else `100`.
- `hexInput` ref: synced from `hexValue` (for text input editing without overwriting during typing).
- `onColorInput(e)` — called on `<input type="color">` `input` event: gets `e.target.value` (hex). If `hasAlpha`, build rgba with current alpha → `setToken`. Else `setToken(key, hex)`. Does NOT commitTheme (preview only).
- `onColorChange()` — called on `<input type="color">` `change` event: calls `editor.commitTheme()`.
- `onHexInput(e)` — on text input `input` event: validate with `/^#[0-9a-fA-F]{6}$/`. If valid, same logic as `onColorInput`.
- `onHexBlur()` — calls `editor.commitTheme()`.
- `onAlphaInput(e)` — on range `input` event: clamp 0-100, `buildRgba(hexValue, newAlpha)` → `setToken`. Does NOT commit.
- `onAlphaChange()` — on range `change` event: calls `editor.commitTheme()`.

Template:
```html
<div class="color-row">
  <label class="token-label">{{ tokenKey }}</label>
  <div class="color-controls">
    <input type="color" :value="hexValue" @input="onColorInput" @change="onColorChange" class="color-picker" />
    <input type="text" v-model="hexInput" @input="onHexInput" @blur="onHexBlur" class="hex-input" maxlength="7" />
    <template v-if="hasAlpha">
      <input type="range" :value="alphaValue" min="0" max="100" @input="onAlphaInput" @change="onAlphaChange" class="alpha-slider" />
      <span class="alpha-label">{{ alphaValue }}%</span>
```
