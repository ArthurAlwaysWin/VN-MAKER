```
/**
 * assetPath — Environment detection and asset path resolution.
 * Single source of truth for runtime mode and resource path prefix.
 *
 * Environments:
 *   'electron' — window.ipcRenderer exists (Electron main window)
 *   'preview'  — iframe receiving postMessage {type:'start'} from editor
 *   'web'      — standalone browser (itch.io, static server, etc.)
 *
 * Per D-05: This module is the single source of truth for environment and paths.
 * Per D-06: BASE_PATH is 'asset://' for electron/preview, './assets/' for web.
 * Per D-07: resolvePath() handles hardcoded asset:// references.
 */
```

**Exports (all named, no defaults):**

1. `export let ENV = 'web';` — `@type {'electron'|'preview'|'web'}`, default 'web'
2. `export let BASE_PATH = './assets/';` — default for web mode
3. `export let SCRIPT_PATH = './script.json';` — path to script.json (web default); Electron uses `/game/script.json`

4. `export async function detectEnvironment()`:
   - Per D-03: Check `window.ipcRenderer` first → if truthy, set `ENV = 'electron'`, `BASE_PATH = 'asset://'`, `SCRIPT_PATH = '/game/script.json'`, return `'electron'`
   - Per D-04: Check `window.parent !== window` → if true, call `_waitForEditorHandshake()`:
     - If handshake succeeds → set `ENV = 'preview'`, `BASE_PATH = 'asset://'`, `SCRIPT_PATH = '/game/script.json'`, return `'preview'`
   - Default: `ENV = 'web'`, `BASE_PATH = './assets/'`, `SCRIPT_PATH = './script.json'`, return `'web'`

5. `export function resolvePath(path)`:
   - If `!path` → return `''`
   - If `path.startsWith('http://') || path.startsWith('https://') || path.startsWith('data:')` → return `path` (pass through)
   - If `path.startsWith('asset://')` → return `BASE_PATH + path.slice(8)` (8 = 'asset://'.length)
   - If `path.startsWith('/game/')` → return `BASE_PATH + path.slice(6)` (6 = '/game/'.length)
   - Otherwise → return `BASE_PATH + path` (bare relative path)

6. Private `function _waitForEditorHandshake()`:
   - Returns `Promise<boolean>`
   - CRITICAL: Must send `window.parent.postMessage({ type: 'ready' }, '*')` BEFORE starting timeout listener — per existing ready→start handshake protocol in main.js line 884
   - Listen for `message` event where `e.data?.type === 'start'`
   - Timeout: 200ms (agent discretion) — resolves `false` if no start message received
   - On match: clear timeout, remove listener, resolve `true`
   - On timeout: remove listener, resolve `false`

**CRITICAL — handshake coexistence:** The `_waitForEditorHandshake` only peeks at whether a `start` message arrives within the timeout window. It does NOT consume the message — the actual `start` handler in `initPreview()` (main.js) will still process it. The detection adds a one-shot listener that self-removes after resolving. The `start` event will bubble to other listeners normally since we don't call `e.stopPropagation()`.

Wait — actually, re-reading the research note on Pitfall 2 more carefully: The editor sends `start` after receiving `ready`. But in the new flow, `detectEnvironment()` runs before `initPreview()` sets up its `start` handler. So the detection's `_waitForEditorHandshake` sends `ready`, the editor responds with `start`, and the detection listener catches it. Then `initPreview()` sets up its own `start` listener. But by then `start` was already sent by the editor.

**Solution:** The `_waitForEditorHandshake` listener, upon receiving `start`, should store `e.data` on a module-level variable so `initPreview()` can read it later. Add:

```javascript
/** @type {MessageEvent['data']|null} Captured start message from handshake detection */
export let _capturedStartMsg = null;
```

In the handler, set `_capturedStartMsg = e.data` before resolving. In Plan 02, `initPreview()` will check `_capturedStartMsg` before setting up the message listener — if it exists, process it immediately instead of waiting.

**Code style:** 2-space indent, single quotes, semicolons, named exports only, underscore prefix on private functions, section divider comments using `// ─── Section Name ───────`.

```
/**
 * WebSaveManager — IndexedDB-based save/load for standalone Web mode.
 *
 * Drop-in replacement for SaveManager with identical async public API.
 * Uses IndexedDB instead of Electron IPC. 108 regular slots + 1 quicksave.
 * No thumbnails in Web mode per D-02 — hasThumbnail is always false.
 *
 * Per D-01: IndexedDB backend, 108 slots + 1 quicksave, matching Electron version.
 * Per D-02: No thumbnails — text-only preview info (scene, dialogue, timestamp).
 */
```

**Class: `export class WebSaveManager`**

**Constructor:**
```javascript
constructor() {
  /** @type {number} Maximum save slots */
  this.slotCount = 108;
  /** @type {IDBDatabase|null} */
  this._db = null;
  /** @type {Map<number|string, Object>} In-memory cache */
  this._cache = new Map();
  /** @type {boolean|undefined} Quick save existence (lazy) */
  this._hasQuickSave = undefined;
  /** @type {number} Compat with SaveManager */
  this._lastMigrationCount = 0;
}
```

**Private helpers (4 methods):**

1. `async _getDb()` — Opens IndexedDB database named `'galgame-saves'`, version 1. In `onupgradeneeded`: create object store `'saves'` with `{ keyPath: 'slot' }`. Cache `this._db`. Return `IDBDatabase`.

2. `_put(db, storeName, record)` — Returns `Promise<void>`. Creates `readwrite` transaction, calls `put(record)`, resolves on `tx.oncomplete`, rejects on `tx.onerror`.

3. `_get(db, storeName, key)` — Returns `Promise<Object|null>`. Creates `readonly` transaction, calls `get(key)`, resolves with `request.result || null` on `request.onsuccess`, rejects on `request.onerror`.

4. `_delete(db, storeName, key)` — Returns `Promise<void>`. Creates `readwrite` transaction, calls `delete(key)`, resolves on `tx.oncomplete`, rejects on `tx.onerror`.

5. `_getAll(db, storeName)` — Returns `Promise<Array>`. Creates `readonly` transaction, calls `getAll()`, resolves with `request.result` on `request.onsuccess`, rejects on `request.onerror`.

**Public API (mirrors SaveManager exactly — 8 methods):**

1. `async save(slot, state, previewText, thumbnail = null)`:
   - `thumbnail` parameter accepted but ignored (D-02 — no thumbnails in Web mode)
   - Deep-clone state: `JSON.parse(JSON.stringify(state))`
   - Truncate history to 50: `if (plainState.history?.length > 50) plainState.history = plainState.history.slice(-50);`
   - Build record: `{ slot, version: 1, state: plainState, previewText: previewText || '', sceneName: plainState.currentScene || '', timestamp: Date.now(), date: new Date().toLocaleString('zh-CN'), hasThumbnail: false }`
   - `try { const db = await this._getDb(); await this._put(db, 'saves', record); this._cache.set(slot, { slot, previewText, sceneName, timestamp, date, hasThumbnail: false }); return { success: true }; } catch (e) { return { success: false, error: e.message }; }`

2. `async load(slot)`:
   - `const db = await this._getDb(); const record = await this._get(db, 'saves', slot);`
   - If `!record`: `console.error('[WebSaveManager] Slot not found:', slot); return null;`
   - Return `{ version: record.version, state: record.state, previewText: record.previewText, sceneName: record.sceneName, timestamp: record.timestamp, date: record.date }`

3. `async delete(slot)`:
   - `try { const db = await this._getDb(); await this._delete(db, 'saves', slot); this._cache.delete(slot); return { success: true }; } catch (e) { return { success: false, error: e.message }; }`

4. `async getAllSlots()`:
   - `const db = await this._getDb(); const all = await this._getAll(db, 'saves');`
   - Filter out quicksave: `const regular = all.filter(r => r.slot !== 'quick');`
   - Rebuild cache from regular records
   - Return metadata array: `regular.map(r => ({ slot: r.slot, previewText: r.previewText, sceneName: r.sceneName, timestamp: r.timestamp, date: r.date, hasThumbnail: false }))`

5. `async hasAnySave()`:
   - `const slots = await this.getAllSlots(); return slots.length > 0;`

6. `async quickSave(state, previewText, thumbnail = null)`:
   - Same as `save()` but with `slot: 'quick'` (string key — per Pitfall 4 in research)
   - On success: `this._hasQuickSave = true;`

7. `async quickLoad()`:
   - `const db = await this._getDb(); const record = await this._get(db, 'saves', 'quick');`
   - If `!record`: `console.error('[WebSaveManager] No quicksave found'); return null;`
   - Return same shape as `load()`

8. `async hasQuickSave()`:
   - If `this._hasQuickSave !== undefined` return cached value
   - `const db = await this._getDb(); const record = await this._get(db, 'saves', 'quick');`
   - `this._hasQuickSave = record !== null; return this._hasQuickSave;`

**Error handling:** All public methods use try/catch and return `{ success: boolean, error?: string }` for write operations, or `null` for read operations on failure — matching SaveManager's patterns exactly.
