# Phase 41 Plan 2: Expression Deletion Reference Check + Confirmation + Batch Replace

<objective>
Implement safe expression deletion with pre-deletion reference scanning across all scenes, a confirmation dialog listing affected pages, and batch replacement of stale references with the first remaining expression (D-05/D-06). This prevents stale references from ever being created during normal editor use, complementing the defensive fallbacks added in Plan 1.

Purpose: D-07/D-08 (Plan 1) are last-resort safety nets for stale references. This plan prevents stale references at the source — when the user deletes an expression, all existing references are proactively updated before the expression data is removed. This is the primary UI-04 prevention mechanism.

Output: Two new store methods (`findExpressionReferences`, `replaceExpressionReferences`) in script.js, rewritten `deleteExpression()` in CharacterEditor.vue.

**1. `findExpressionReferences(charId, exprName)`** — Scan ALL scenes and pages for references to a specific character expression. Returns an array of `{ sceneId, sceneName, pageIndex }` objects.

```javascript
/**
 * Find all pages that reference a specific expression for a character.
 * Checks both page-level character expressions and dialogue-level expressions.
 * @param {string} charId - Character ID
 * @param {string} exprName - Expression name to search for
 * @returns {Array<{sceneId: string, sceneName: string, pageIndex: number}>}
 */
function findExpressionReferences(charId, exprName) {
  const refs = [];
  const scenes = data.value?.scenes || {};
  for (const [sceneId, scene] of Object.entries(scenes)) {
    for (let i = 0; i < (scene.pages || []).length; i++) {
      const page = scene.pages[i];
      const charMatch = (page.characters || []).some(
        c => c.id === charId && c.expression === exprName
      );
      const dlgMatch = (page.dialogues || []).some(
        d => d.speaker === charId && d.expression === exprName
      );
      if (charMatch || dlgMatch) {
        refs.push({ sceneId, sceneName: scene.name || sceneId, pageIndex: i });
      }
    }
  }
  return refs;
}
```

**2. `replaceExpressionReferences(charId, oldExpr, newExpr)`** — Batch-replace all references. Mutates data in-place. Does NOT call `pushState()` — caller is responsible for calling it once after all mutations (Pitfall 4: single undo step).

```javascript
/**
 * Batch-replace all references to an expression across all scenes/pages.
 * Replaces in both page-level character data and dialogue-level expressions.
 * Does NOT call pushState() — caller must do so after completing all mutations.
 * @param {string} charId - Character ID
 * @param {string} oldExpr - Expression name to replace
 * @param {string} newExpr - Replacement expression name (or null)
 */
function replaceExpressionReferences(charId, oldExpr, newExpr) {
  const scenes = data.value?.scenes || {};
  for (const scene of Object.values(scenes)) {
    for (const page of (scene.pages || [])) {
      for (const char of (page.characters || [])) {
        if (char.id === charId && char.expression === oldExpr) {
          char.expression = newExpr;
        }
      }
      for (const dlg of (page.dialogues || [])) {
        if (dlg.speaker === charId && dlg.expression === oldExpr) {
          dlg.expression = newExpr;
        }
      }
    }
  }
}
```

Replace the entire `deleteExpression` function with:

```javascript
/**
 * Delete an expression with reference check and batch replacement (D-05).
 * If the expression is referenced in any page, shows a confirmation dialog
 * listing affected pages. On confirmation, batch-replaces all references
 * with the first remaining expression, then deletes.
 * @param {string} exprName - Expression name to delete
 */
async function deleteExpression(exprName) {
  if (!selectedChar.value) return;

  // D-05: Check all scenes for references to this expression
  const refs = script.findExpressionReferences(selectedId.value, exprName);

  if (refs.length === 0) {
    // No references — simple confirmation (existing UX)
    if (!confirm(`确定要删除表情「${exprName}」吗？\n图片文件也会从磁盘删除。`)) return;
  } else {
    // Has references — build detailed confirmation message (D-05)
    const pageList = formatReferenceList(refs);
    const msg = `您在 ${pageList} 使用了该角色表情差分，是否仍然删除该表情？\n\n删除后，将自动把所有引用替换为第一个表情。\n图片文件也会从磁盘删除。`;
    if (!confirm(msg)) return;

    // D-06: Replacement target is first expression that isn't the one being deleted
    const replacement = Object.keys(selectedChar.value.expressions)
      .find(k => k !== exprName) || null;

    // Batch replace all references (does NOT call pushState — Pitfall 4)
    script.replaceExpressionReferences(selectedId.value, exprName, replacement);
  }

  // Delete expression from character data
  const exprPath = selectedChar.value.expressions[exprName];
  delete selectedChar.value.expressions[exprName];

  // Single pushState for entire operation (D-05 batch = one undo step)
  script.pushState();

  // Delete asset file from disk (async, after state committed)
  if (exprPath) {
    const filename = exprPath.split('/').pop();
    await assets.deleteAsset('characters', filename);
  }
}
```

**Add helper function** `formatReferenceList(refs)` above `deleteExpression()` to format the confirmation message. Shows up to 10 page references, then summarizes the rest:

```javascript
/**
 * Format a list of expression references for the confirmation dialog.
 * Groups by scene, shows page numbers. Truncates at 10 entries.
 * @param {Array<{sceneId: string, sceneName: string, pageIndex: number}>} refs
 * @returns {string} Formatted reference list (e.g., "「场景A」第1,3页、「场景B」第2页")
 */
function formatReferenceList(refs) {
  // Group by scene
  const grouped = {};
  for (const ref of refs) {
    if (!grouped[ref.sceneId]) {
      grouped[ref.sceneId] = { name: ref.sceneName, pages: [] };
    }
    grouped[ref.sceneId].pages.push(ref.pageIndex + 1); // 1-based for display
  }

  const parts = [];
  let count = 0;
  for (const { name, pages } of Object.values(grouped)) {
    if (count >= 10) {
      parts.push(`...及其他 ${refs.length - count} 处`);
      break;
    }
    const pageNums = pages.slice(0, 10 - count).join(', ');
    parts.push(`「${name}」第 ${pageNums} 页`);
    count += pages.length;
  }
  return parts.join('、');
}
```

Key implementation notes:
- Uses native `confirm()` per project convention (10+ existing usages, NOT a custom modal)
- Replacement computed BEFORE deletion: `Object.keys().find(k => k !== exprName)` gives the first key that isn't being deleted — equivalent to `Object.keys()[0]` after deletion per D-06
- `replaceExpressionReferences` mutates in-place without pushState → then we delete → then ONE `pushState()` = single undo step (Pitfall 4)
- Asset file deletion is async and happens AFTER state is committed — if it fails, the expression is still removed from data (consistent with existing pattern)
- If deleting the LAST expression (replacement = null), references get `expression: null` — D-07/D-08 fallback from Plan 1 handles this gracefully (shows placeholder)

<pitfalls>
1. **Pitfall 4 (from research):** Do NOT call `pushState()` inside `replaceExpressionReferences()`. The entire sequence (batch replace → delete expression) must be a single undo snapshot. Only call `pushState()` once after ALL mutations are complete.
2. **Pitfall 5 (from research):** If deleting the last expression, `Object.keys().find(k => k !== exprName)` returns `undefined` → replacement is `null`. References get `expression: null`. This is handled by D-07/D-08 fallback (Plan 1) — canvas shows placeholder, engine shows nothing. Acceptable edge case.
3. **Replacement computation order:** Compute replacement BEFORE deleting the expression from the dict. After `delete expressions[exprName]`, `Object.keys()[0]` would already exclude it, but computing before is clearer and safer.
</pitfalls>
