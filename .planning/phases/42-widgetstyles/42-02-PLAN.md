    Create `src/ui/widgets/TabWidget.js` with JSDoc file header:
    ```
    /**
     * TabWidget — creates a tab bar from widgetStyles.tab config.
     * Supports 5 shape variants: rectangle, pill, underline, trapezoid, ribbon.
     * Each shape uses different CSS/DOM to achieve its visual style.
     */
    ```

    Export function `createTabBar(labels, config, onSelect)`:
    - `labels`: string array of tab labels (e.g. `['声音', '画面', '游戏']`)
    - `config`: merged widgetStyles.tab object (shape, activeColor, inactiveColor, etc.)
    - `onSelect`: callback `(index) => void` called when a tab is clicked
    - Returns `{ el, setActive(index) }` — el is the container div, setActive updates visual state

    Implementation:
    1. Create container: `div.gm-tab-bar` with `display: flex; gap: 4px; align-items: flex-end;`
    2. For each label, create a `button.gm-tab` element
    3. Apply per-shape styling using a `_applyTabShape(btn, shape, isActive, config)` helper:

    **shape='rectangle'** (default):
    - No border-radius
    - Active: `background: config.activeColor; color: config.activeTextColor`
    - Inactive: `background: config.inactiveColor; color: config.inactiveTextColor`
    - Base styles: `padding: 8px 20px; border: none; cursor: pointer; font-size: ${config.fontSize}px`
    - If config.fontFamily and sanitizeCssValue passes: set fontFamily

    **shape='pill'**:
    - Same as rectangle but add: `border-radius: 999px`

    **shape='underline'**:
    - Background: transparent for both states
    - Active: `color: config.activeTextColor; border-bottom: 3px solid ${config.activeColor}`
    - Inactive: `color: config.inactiveTextColor; border-bottom: 3px solid transparent`
    - padding-bottom: 6px (extra space for border)

    **shape='trapezoid'**:
    - Same base as rectangle
    - Add: `clip-path: polygon(12px 0%, calc(100% - 12px) 0%, 100% 100%, 0% 100%)`
    - Slightly wider horizontal padding: `padding: 8px 28px` (compensate for clipped edges)

    **shape='ribbon'**:
    - If `config.nineSlice?.src` exists: set background transparent, create child nineSlice div (same pattern as PanelWidget — position:absolute, inset:0, z-index:-1, border-image from nineSlice config, resolve path with resolvePath). Set btn position:relative, isolation:isolate, overflow:hidden.
    - If no nineSlice: use clip-path: `polygon(0% 0%, calc(100% - 16px) 0%, 100% 50%, calc(100% - 16px) 100%, 0% 100%)` with padding: `8px 32px 8px 16px`
    - Active/inactive colors same as rectangle

    4. If `config.activeBackgroundImage` is truthy and passes sanitizeCssValue: for active tabs, set `background-image: url("${resolvePath(config.activeBackgroundImage)}")` with `background-size: cover; background-position: center`

    5. Click handler on each button:
    - Call `setActive(clickedIndex)` (internal)
    - Call `onSelect(clickedIndex)`

    6. `setActive(index)` function:
    - Iterate all tab buttons, apply inactive styles
    - Apply active styles to button at `index`
    - Store `_activeIndex = index` on container for external query

    7. Initial state: first tab (index 0) is active

    Imports: `sanitizeCssValue` from `'../sanitize.js'`, `resolvePath` from `'../../engine/assetPath.js'`

    **Part A: Create `src/ui/widgets/ToggleWidget.js`**

    JSDoc file header:
    ```
    /**
     * ToggleWidget — creates toggle controls from widgetStyles.toggle config.
     * Supports 4 style variants: pill, radio, checkbox, button-pair.
     * Each style produces completely different DOM structure.
     */
    ```

    Export function `createToggle(id, config, value, onChange)`:
    - `id`: unique string identifier for this toggle (used for radio name attribute)
    - `config`: merged widgetStyles.toggle object
    - `value`: initial boolean value (true = ON)
    - `onChange`: callback `(newValue: boolean) => void`
    - Returns `{ el, setValue(bool) }` — el is the toggle container, setValue programmatically updates

    Dispatch on `config.style` to create different DOM:

    **style='pill'** (default):
    DOM (matches spec Section 7.3):
    ```html
    <div class="gm-toggle gm-toggle-pill" data-on="{value}">
      <div class="gm-toggle-thumb"></div>
      <span class="gm-toggle-label-on">{config.onLabel}</span>
      <span class="gm-toggle-label-off">{config.offLabel}</span>
    </div>
    ```
    Inline styles on container:
    - width: `${config.width}px`, height: `${config.height}px`
    - borderRadius: `${config.height / 2}px` (capsule)
    - position: relative, cursor: pointer, overflow: hidden
    - display: flex, alignItems: center
    - background: value ? config.onColor : config.offColor (use sanitizeCssValue)
    - fontSize: `${config.fontSize}px`

    Thumb styles:
    - width/height: `${config.height - 4}px` (2px margin each side)
    - borderRadius: 50%
    - background: config.thumbColor
    - position: absolute
    - left: value ? `${config.width - config.height + 2}px` : '2px' (slide position)
    - transition: 'left 0.15s ease' (minimal, per decision: no animations → keep very brief for usability)

    Label styles (ON/OFF text):
    - position: absolute, top: 50%, transform: translateY(-50%)
    - ON label: left 8px, display: value ? '' : 'none'
    - OFF label: right 8px, display: value ? 'none' : ''
    - color: '#fff', fontSize from config

    Click handler: toggle value, update data-on attribute, reposition thumb, swap label visibility, update background color, call onChange

    **style='radio'**:
    DOM (matches spec Section 7.3):
    ```html
    <div class="gm-toggle gm-toggle-radio">
      <label><input type="radio" name="{id}" value="on" {checked if value}> <span class="gm-radio-custom"></span> {config.onLabel}</label>
      <label><input type="radio" name="{id}" value="off" {checked if !value}> <span class="gm-radio-custom"></span> {config.offLabel}</label>
    </div>
    ```
    Container: display: flex, gap: 12px, fontSize from config
    Radio custom span: 12px circle, border: 2px solid rgba(255,255,255,0.4). When checked: background = config.onColor, border-color = config.onColor
    Hide native radio input: position: absolute, opacity: 0, width: 0, height: 0
    Label: cursor: pointer, display: flex, alignItems: center, gap: 4px, color: '#fff'
    Change handler on inputs: update checked state, call onChange with (radio.value === 'on')

    **style='checkbox'**:
    DOM:
    ```html
    <div class="gm-toggle gm-toggle-checkbox">
      <label><input type="checkbox" {checked if value}> <span class="gm-checkbox-custom"></span> {config.onLabel}</label>
    </div>
    ```
    Checkbox custom span: 18px square, border: 2px solid rgba(255,255,255,0.4), borderRadius: 3px. When checked: background = config.onColor, border-color = config.onColor, add checkmark via textContent '✓' (color: #fff, centered)
    Hide native checkbox: position: absolute, opacity: 0, width: 0, height: 0
    Label: cursor: pointer, display: flex, alignItems: center, gap: 8px, color: '#fff', fontSize from config
    Change handler: update visual, call onChange with checkbox.checked

    **style='button-pair'**:
    DOM (matches spec Section 7.3):
    ```html
    <div class="gm-toggle gm-toggle-btnpair">
      <button class="gm-toggle-btn gm-btn-on {active if value}">{config.onLabel}</button>
      <button class="gm-toggle-btn gm-btn-off {active if !value}">{config.offLabel}</button>
    </div>
    ```
    Container: display: flex, gap: 2px
    Button styles: padding: 6px 16px, border: none, cursor: pointer, fontSize from config, borderRadius: 4px
    Active button: background = config.onColor, color = '#fff'
    Inactive button: background = config.offColor, color = 'rgba(255,255,255,0.6)'
    Click handlers: toggle active class, call onChange

    `setValue(bool)` method: update DOM state without triggering onChange (for programmatic updates)

    Import: `sanitizeCssValue` from `'../sanitize.js'`

    **Part B: Create `src/ui/widgets/SliderWidget.js`**

    JSDoc file header:
    ```
    /**
     * SliderWidget — creates slider controls from widgetStyles.slider config.
     * Supports configurable track/fill/thumb colors, sizes, shapes,
     * and optional thumb/track images.
     */
    ```

    Export function `createSlider(config, value, min, max, step, onChange)`:
    - `config`: merged widgetStyles.slider object
    - `value`: initial numeric value
    - `min`, `max`, `step`: slider range constraints
    - `onChange`: callback `(newValue: number) => void`
    - Returns `{ el, setValue(number), getValue() }` — el is the slider container

    Implementation:
    1. Create container `div.gm-slider` with position: relative, display: flex, alignItems: center
    2. Create `input[type=range].gm-slider-input` with min, max, step, value
    3. Set CSS custom properties on the input for styling:
       - `--gm-track-color`: `sanitizeCssValue(config.trackColor) || 'rgba(255,255,255,0.15)'`
       - `--gm-fill-color`: `sanitizeCssValue(config.fillColor) || 'rgba(180,160,255,0.8)'`
       - `--gm-thumb-color`: `sanitizeCssValue(config.thumbColor) || '#fff'`
       - `--gm-thumb-size`: `${config.thumbSize || 16}px`
       - `--gm-track-height`: `${config.trackHeight || 4}px`
       - `--gm-thumb-radius`: config.thumbStyle === 'square' ? '2px' : '50%'
    4. Set input width to 100%, appearance: none, background: transparent, cursor: pointer
    5. If `config.thumbImage` is truthy:
       - Set `--gm-thumb-image`: `url("${resolvePath(config.thumbImage)}")`
    6. If `config.trackImage` is truthy:
       - Set `--gm-track-image`: `url("${resolvePath(config.trackImage)}")`
    7. Input event handler: call `onChange(Number(input.value))`
    8. `setValue(n)`: set input.value = n (no onChange trigger)
    9. `getValue()`: return Number(input.value)

    Also export function `getSliderCSS()`:
    - Returns a CSS string that should be injected once into the document for slider styling
    - This CSS targets `.gm-slider-input` with:
      - `::-webkit-slider-runnable-track` — height from var(--gm-track-height), background from var(--gm-track-color), border-radius 2px
      - `::-webkit-slider-thumb` — appearance:none, width/height from var(--gm-thumb-size), background from var(--gm-thumb-color), border-radius from var(--gm-thumb-radius), margin-top calculated, cursor pointer
      - `::-moz-range-track` — same as webkit track
      - `::-moz-range-thumb` — same as webkit thumb, border:none
      - `::-moz-range-progress` — background from var(--gm-fill-color) (Firefox fill)
    - Note: Webkit fill color is achieved via background: linear-gradient on track (calculate fill percentage from value) — update on input event

    Import: `sanitizeCssValue` from `'../sanitize.js'`, `resolvePath` from `'../../engine/assetPath.js'`
