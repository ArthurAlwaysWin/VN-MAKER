# Plan 43-02: BacklogScreen.setLayout()

## Goal
Add `setLayout(config)` to BacklogScreen that applies background/header/entry configuration
from `ui.backlogScreen` schema. When config is null, existing rendering is unchanged.

## Context
- BacklogScreen.js: 134 lines, rebuilds DOM in show()
- Design spec Section 5.3 defines complete JSON schema
- Pattern: SettingsScreen.setLayout() stores config, show() branches on it
- sanitize.js: sanitizeCssValue for safe CSS
- assetPath.js: resolvePath for backgroundImage paths

## Design Spec Schema (from Section 5.3)
```json
{
  "background": "rgba(0,0,0,0.85)",
  "backgroundImage": null,
  "header": {
    "title": "回 想",
    "backgroundImage": null,
    "height": 60
  },
  "entry": {
    "speakerColor": null,
    "speakerFontSize": 13,
    "textFontSize": 15,
    "background": "transparent",
    "hoverBackground": "rgba(255,255,255,0.05)",
    "borderBottom": "1px solid rgba(255,255,255,0.06)",
    "padding": [12, 20]
  }
}
```

## Tasks

### Task 1: Add setLayout() method and config storage
**File:** `src/ui/BacklogScreen.js`
**Action:** Add after constructor:
```js
setLayout(config) {
  this._layoutConfig = config || null;
}
```
Add `this._layoutConfig = null;` in constructor.

### Task 2: Config-driven show() — screen background
**File:** `src/ui/BacklogScreen.js`
**Action:** After `this.el.innerHTML = ...`, if config:
- `cfg.background` → `this.el.style.background`
- `cfg.backgroundImage` → `this.el.style.backgroundImage` via resolvePath

### Task 3: Config-driven show() — header
**File:** `src/ui/BacklogScreen.js`
**Action:** After header HTML is set:
- `cfg.header.title` → replace '回 想' title text
- `cfg.header.backgroundImage` → header div background-image
- `cfg.header.height` → header div height in px

### Task 4: Config-driven show() — entry styles
**File:** `src/ui/BacklogScreen.js`
**Action:** In the history.forEach loop, apply entry config:
- `cfg.entry.speakerColor` → speakerDiv.style.color (if set, overrides character color)
- `cfg.entry.speakerFontSize` → speakerDiv.style.fontSize
- `cfg.entry.textFontSize` → textDiv.style.fontSize
- `cfg.entry.background` → div.style.background
- `cfg.entry.hoverBackground` → add mouseenter/mouseleave listeners
- `cfg.entry.borderBottom` → div.style.borderBottom
- `cfg.entry.padding` → div.style.padding (array → `${v[0]}px ${v[1]}px`)

Note: speakerColor from config takes priority over character.color when both exist.

### Task 5: Unit tests
**File:** `tests/backlogScreenLayout.test.js`
**Action:** Create tests:
1. setLayout(null) → _layoutConfig is null
2. setLayout(config) → _layoutConfig stores config
3. show() with null config → no style changes (COMPAT-02)
4. show() with config.background → el.style.background set
5. show() with config.header.title → header text changed
6. show() with config.entry.speakerColor → overrides character color

## COMPAT-02 Verification
The null path must produce identical output:
- `show()`: if (!this._layoutConfig) → existing innerHTML template unchanged
- Entry loop: if (!this._layoutConfig) → existing speaker/text div creation unchanged
- Character color logic: if no config.entry.speakerColor → charColor still applied as before
