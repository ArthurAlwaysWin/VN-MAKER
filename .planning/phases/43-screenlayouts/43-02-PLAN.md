Per SCREEN-02 and COMPAT-02 (CONTEXT decisions #1-#4: setLayout method, store-on-set/apply-on-show, inline defaults, config branching in show()).

1. ADD IMPORTS at top (before class declaration):
   import { sanitizeCssValue, clampField } from './sanitize.js';
   import { resolvePath } from '../engine/assetPath.js';

2. ADD FIELD in constructor (after this.container.appendChild(this.el)):
   this._layoutConfig = null;

3. ADD setLayout() METHOD between constructor and show():
   setLayout(config) { this._layoutConfig = config || null; }
   Include JSDoc: @param {object|null} config — null preserves defaults (COMPAT-02)

4. MODIFY show() — config-aware title and screen styling:
   - At top: const cfg = this._layoutConfig;
   - Compute title: const title = cfg?.header?.title ?? '回 想';
   - Replace hardcoded '回 想' in innerHTML with ${title}
   - AFTER innerHTML and close-button listener, add:
     if (cfg) {
       - Background color: sanitizeCssValue(cfg.background) -> this.el.style.background
       - Background image: sanitizeCssValue(cfg.backgroundImage) + resolvePath -> child div layer
         (position absolute, inset 0, zIndex 0, backgroundImage url(...), backgroundSize cover,
         backgroundPosition center, pointerEvents none); insert before firstChild
       - Header bg image: sanitizeCssValue(cfg.header.backgroundImage) + resolvePath ->
         header.style.backgroundImage with cover/center
       - Header height: clampField('height', cfg.header.height) -> header.style.minHeight
     }

5. MODIFY ENTRY LOOP (history.forEach) — apply entry config:
   - At top of forEach body: const eCfg = cfg?.entry;

   - FOR SPEAKER DIV (inside if (entry.speakerName) block, after creating speakerDiv):
     if (eCfg?.speakerFontSize) -> clampField('fontSize', ...) -> speakerDiv.style.fontSize
     if (!charColor and eCfg?.speakerColor) -> sanitizeCssValue -> speakerDiv.style.color
     (This is a FALLBACK — character-specific color takes priority, config is default)

   - FOR TEXT DIVS (both branches — with speaker and italic narration):
     if (eCfg?.textFontSize) -> clampField('fontSize', ...) -> textDiv.style.fontSize

   - FOR ENTRY DIV (after all children appended, after voice button logic):
     if (eCfg) {
       - background: sanitizeCssValue(eCfg.background) -> div.style.background
       - borderBottom: sanitizeCssValue(eCfg.borderBottom) -> div.style.borderBottom
       - padding: Array [v, h] -> clampField('padding', v) + clampField('padding', h) ->
         div.style.padding = v + 'px ' + h + 'px'
       - hoverBackground: sanitizeCssValue(eCfg.hoverBackground) -> mouseenter/mouseleave handlers
         (store origBg = div.style.background || ''; mouseenter sets hoverBg; mouseleave restores)
     }

COMPAT-02 GUARANTEE: When _layoutConfig is null:
- cfg is null; title ?? '回 想' (identical)
- if (cfg) block skipped — no style overrides
- eCfg undefined — no entry overrides
- Speaker color: !charColor and eCfg?.speakerColor = false
- No hover handlers attached; CSS :hover from stylesheet works normally
- Zero rendering difference
