# Plan 43-03: GameMenu.setLayout()

## Goal
Add `setLayout(config)` to GameMenu that applies position/width/background/borderRadius/
backdropBlur/buttonGap/button text+icon configuration from `ui.gameMenu` schema.
When config is null, existing rendering is unchanged.

## Context
- GameMenu.js: 72 lines, _render() called once in constructor, buttons use data-action delegation
- Design spec Section 5.2 defines complete JSON schema
- Pattern: SettingsScreen.setLayout() stores config, show() branches on it
- sanitize.js: sanitizeCssValue + clampField for safe CSS
- assetPath.js: resolvePath for backgroundImage paths

## Design Spec Schema (from Section 5.2)
```json
{
  "position": "center",
  "width": 260,
  "background": "rgba(0,0,0,0.75)",
  "backgroundImage": null,
  "borderRadius": 8,
  "backdropBlur": 12,
  "buttonGap": 8,
  "buttons": {
    "save":     { "text": "存 档", "icon": null },
    "load":     { "text": "読 档", "icon": null },
    "backlog":  { "text": "回 想", "icon": null },
    "settings": { "text": "設 定", "icon": null },
    "title":    { "text": "返回标题", "icon": null },
    "close":    { "text": "返 回", "icon": null }
  }
}
```

## Tasks

### Task 1: Add setLayout() method and config storage
**File:** `src/ui/GameMenu.js`
**Action:** Add after constructor:
```js
setLayout(config) {
  this._layoutConfig = config || null;
  this._render(); // Re-render with new config
}
```
Add `this._layoutConfig = null;` in constructor (before `this._render()`).

Note: GameMenu._render() is called once in constructor. setLayout() triggers re-render
to apply config. Since _render() rebuilds innerHTML completely, this is safe.

### Task 2: Config-driven _render() — panel styles
**File:** `src/ui/GameMenu.js`
**Action:** After building innerHTML with buttons, if config:
- `cfg.position` → panel positioning (center=margin auto, left=align-self start, right=align-self end)
- `cfg.width` → panel.style.width
- `cfg.background` → panel.style.background
- `cfg.backgroundImage` → panel.style.backgroundImage via resolvePath
- `cfg.borderRadius` → panel.style.borderRadius
- `cfg.backdropBlur` → panel.style.backdropFilter
- `cfg.buttonGap` → panel.style.gap (flex column gap)

### Task 3: Config-driven _render() — button text and icons
**File:** `src/ui/GameMenu.js`
**Action:** Instead of hardcoded button HTML, generate from config:
- Default button labels: { save: '存 档', load: '読 档', backlog: '回 想', settings: '設 定', title: '返回标题', close: '返 回' }
- If `cfg.buttons[action].text` → use that text
- If `cfg.buttons[action].icon` → prepend `<img src="${resolvePath(icon)}" class="game-menu-icon" />` to button

### Task 4: Unit tests
**File:** `tests/gameMenuLayout.test.js`
**Action:** Create tests:
1. setLayout(null) → _layoutConfig is null
2. setLayout(config) → _layoutConfig stores config, re-renders
3. _render with null config → hardcoded button text unchanged (COMPAT-02)
4. _render with config.background → panel.style.background set
5. _render with config.buttons.save.text → save button shows custom text
6. _render with config.buttons.save.icon → save button has icon img

## COMPAT-02 Verification
The null path must produce identical output:
- `_render()`: if (!this._layoutConfig) → existing innerHTML template with hardcoded Chinese labels
- Panel: no style changes applied
- Buttons: same 6 buttons with same data-action attributes and hardcoded text
- Event delegation: same click handler structure unchanged
