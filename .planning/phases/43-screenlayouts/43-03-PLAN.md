  <action>
Per SCREEN-03 and COMPAT-02 (CONTEXT decisions #1-#4: setLayout method, store-on-set/apply-on-show, inline defaults, config branching in _render).

1. ADD IMPORTS at top (before class declaration):
   import { sanitizeCssValue, clampField, clamp } from './sanitize.js';
   import { resolvePath } from '../engine/assetPath.js';

2. ADD FIELD in constructor (after callback declarations, before _render):
   this._layoutConfig = null;

3. RESTRUCTURE CONSTRUCTOR — move click event listener from _render() to constructor.
   This prevents double-attach when show() calls _render() again with config.
   
   Keep this._render() call in constructor.
   Move the this.el.addEventListener('click', ...) block from _render() to constructor,
   placed AFTER this._render(). The event delegation uses e.target.closest('[data-action]')
   so it works regardless of innerHTML rebuilds. Same switch logic: save/load/backlog/settings/title
   actions fire callbacks; close is handled by this.hide() called before the switch.

4. ADD setLayout() METHOD after toggle():
   setLayout(config) { this._layoutConfig = config || null; }
   Include JSDoc: @param {object|null} config — null preserves defaults (COMPAT-02)

5. MODIFY show() — re-render when config exists:
   show() {
     if (this._layoutConfig) { this._render(); }
     this.el.classList.remove('hidden');
     requestAnimationFrame(() => this.el.classList.add('visible'));
   }
   When null: show() just toggles visibility (identical to current — COMPAT-02).

6. REWRITE _render() — programmatic buttons with config support:
   _render() {
     const cfg = this._layoutConfig;
     
     Define btnDefs array — 6 entries, each { action, text } where text uses
     cfg?.buttons?.[action]?.text ?? hardcodedDefault. Hardcoded defaults must match
     current values exactly: '存 档', '读 档', '回 想', '设 定', '返回标题', '返 回'.
     
     Create panel div (className 'game-menu-panel').
     For each btnDef: create button (className 'game-menu-button', dataset.action = action).
       - If cfg?.buttons?.[action]?.icon: sanitizeCssValue -> resolvePath -> create img element
         (width/height 20px, marginRight 8px, verticalAlign middle, draggable false), append to btn.
       - Append text via document.createTextNode(def.text)
       - Append btn to panel.
     
     this.el.innerHTML = ''; this.el.appendChild(panel);
     
     if (cfg) {
       - Overlay bg: sanitizeCssValue(cfg.background) -> this.el.style.background
       - Overlay blur: clamp(cfg.backdropBlur, 0, 50) -> this.el.style.backdropFilter
       - Background image: sanitizeCssValue(cfg.backgroundImage) + resolvePath -> child div layer
         (absolute, inset 0, zIndex 0, cover/center, pointerEvents none); insertBefore firstChild;
         set panel position relative + zIndex 1
       - Panel width: clampField('width', cfg.width) -> panel.style.width; panel.style.minWidth = 'auto'
       - Panel borderRadius: clampField('borderRadius', cfg.borderRadius) -> panel.style.borderRadius
       - Button gap: clamp(cfg.buttonGap, 0, 100) -> panel.style.gap
       - Position: if cfg.position !== 'center':
         'top' -> this.el.style.alignItems = 'flex-start'; panel.style.marginTop = '80px'
         'bottom' -> this.el.style.alignItems = 'flex-end'; panel.style.marginBottom = '80px'
     }
   }

COMPAT-02 GUARANTEE: When _layoutConfig is null:
- show() does NOT call _render() — uses constructor's initial render
- In _render(): cfg null, text uses ?? to hardcoded (identical)
- No icons (cfg?.buttons?.*.icon undefined)
- if (cfg) styling block skipped
- Programmatic buttons produce same DOM as current innerHTML
- Click delegation in constructor = same behavior as previous _render() placement
- Zero visual/behavioral difference

NOTE: Moving click handler from _render() to constructor is safe — event delegation on
this.el works across innerHTML rebuilds. Handler reads this.onXxx at call time (current).
