# Plan 01: OKLCH Module + Rule Table + Derivation Function

<objective>
Implement the complete smart color derivation pipeline: OKLCH color space conversion (COLOR-01), declarative rule table for 35 tokens (COLOR-02), and `deriveTokens()` function (COLOR-03). This is a pure engine library with zero coupling to editor UI.
</objective>

<must_haves>
1. `rgbToOklch()` and `oklchToRgb()` roundtrip any sRGB color with ΔE < 1
2. `deriveTokens('#b4a0ff', '#ff6b9d', 'dark')` returns all 35 color token keys with valid rgba/hex/gradient strings
3. `deriveTokens()` produces correct light-mode tokens when mode='light' (text near-black, backgrounds near-white)
4. Out-of-gamut OKLCH values are clamped to valid sRGB via binary-search chroma reduction
5. Zero external dependencies (pure JS)
6. Output token keys exactly match DEFAULT_TOKENS color keys from tokens.js
</must_haves>

<action>
Create `src/engine/oklch.js` with:

1. **sRGB ↔ linear conversion** (same math as contrast.js but self-contained):
   ```
   srgbToLinear(c): c <= 0.04045 ? c/12.92 : ((c+0.055)/1.055)**2.4
   linearToSrgb(c): c <= 0.0031308 ? 12.92*c : 1.055 * c**(1/2.4) - 0.055
   ```

2. **`rgbToOklch(r, g, b)`** — RGB (0-255) → {l, c, h}:
   - Normalize to 0-1, apply srgbToLinear
   - Apply M1 matrix (RGB→LMS):
     ```
     l_ = cbrt(0.4122214708*lr + 0.5363325363*lg + 0.0514459929*lb)
     m_ = cbrt(0.2119034982*lr + 0.6806995451*lg + 0.1073969566*lb)
     s_ = cbrt(0.0883024619*lr + 0.2817188376*lg + 0.6299787005*lb)
     ```
   - Apply M2 matrix (LMS→OKLab):
     ```
     L = 0.2104542553*l_ + 0.7936177850*m_ - 0.0040720468*s_
     a = 1.9779984951*l_ - 2.4285922050*m_ + 0.4505937099*s_
     b = 0.0259040371*l_ + 0.7827717662*m_ - 0.8086757660*s_
     ```
   - Convert Lab→LCH: `C = sqrt(a²+b²)`, `H = atan2(b,a) * 180/π` (normalize to 0-360)
   - Return `{ l: L, c: C, h: H }`

3. **`oklchToRgb(l, c, h)`** — {l, c, h} → [r, g, b] (0-255):
   - LCH→Lab: `a = c*cos(h*π/180)`, `b = c*sin(h*π/180)`
   - Inverse M2 (OKLab→LMS):
     ```
     l_ = l + 0.3963377774*a + 0.2158037573*b
     m_ = l - 0.1055613458*a - 0.0638541728*b
     s_ = l - 0.0894841775*a - 1.2914855480*b
     ```
   - Cube LMS: `lr = l_³, lm = m_³, ls = s_³`
   - Inverse M1 (LMS→linear RGB):
     ```
     R = +4.0767416621*lr - 3.3077115913*lm + 0.2309699292*ls
     G = -1.2684380046*lr + 2.6097574011*lm - 0.3413193965*ls
     B = -0.0041960863*lr - 0.7034186147*lm + 1.7076147010*ls
     ```
   - Apply linearToSrgb, clamp 0-1, scale to 0-255, round

4. **`hexToOklch(hex)`** — '#rrggbb' → {l, c, h}: parse hex, call rgbToOklch

5. **`isInGamut(l, c, h)`** — boolean: convert to RGB, reconvert back, check ΔC < 0.002

6. **`clampChroma(l, c, h)`** — binary-search chroma: bisect [0, c] up to 20 iterations or tolerance 0.001, return max in-gamut chroma

Module style: `@module oklch` JSDoc header, named exports only, zero imports.
</action>
