# Plan 01: Configurable Tab Structure + Icons + Setting Assignment

<objective>
Make the engine settings screen support configurable tabs: custom count/labels (STRUCT-01), optional icons per tab (STRUCT-02), per-tab setting key assignment (STRUCT-03), while preserving backward compatibility when tabBar.tabs is omitted (STRUCT-07).
</objective>

<must_haves>
1. Setting `tabBar.tabs` to a 4-item object array renders 4 tab buttons with custom labels
2. Each tab shows exactly the settings listed in its `settingKeys` array
3. Tab buttons display icon image + text when `tabs[].icon` is set to an asset path
4. Omitting `tabBar.tabs` entirely renders identically to v1.2 (zero visual regression)
5. Setting keys not assigned to any tab are appended to the last tab
6. Legacy string[] format for tabBar.tabs still works (auto-detected and converted)
</must_haves>

### T1: Add `normalizeTabs()` helper + update `_renderStructured()` tab parsing

<read_first>
- src/ui/SettingsScreen.js (lines 1-30 for imports and constants, lines 385-516 for _renderStructured)
- src/engine/settingDefs.js (SETTING_DEFS keys)
- .planning/phases/53-configurable-tabs-engine/53-CONTEXT.md (decisions D-01 through D-04, D-09 through D-12)
</read_first>

<action>
Add a `normalizeTabs(rawTabs)` function near the top of SettingsScreen.js (after the constants) that:

1. If `rawTabs` is null/undefined → return `null` (signals: use defaults)
2. If `rawTabs[0]` is a string → convert each entry to `{ label: entry }` (legacy compat, D-03)
3. If `rawTabs[0]` is an object → return as-is
4. Each tab object has shape: `{ label: string, icon?: string, settingKeys?: string[] }`

Add a `resolveTabSettingKeys(tabs)` function that:

1. Takes the normalized tabs array
2. For each tab: if `settingKeys` is present, use it; otherwise fall back to `SETTING_GROUP_KEYS[index]` (D-04)
3. Collect all assigned keys into a Set
4. Find all SETTING_DEFS keys NOT assigned to any tab → append them to the last tab's keys (D-10)
5. Remove duplicates: if a key appears in multiple tabs, keep only the first occurrence (D-11)
6. Return resolved array of `{ label, icon, settingKeys }` objects

Update `_renderStructured()`:
- Replace `const tabLabels = tabCfg.tabs || DEFAULT_TAB_LABELS;` with:
  ```js
  const rawTabs = tabCfg.tabs;
  const normalized = normalizeTabs(rawTabs);
  const resolvedTabs = normalized
    ? resolveTabSettingKeys(normalized)
    : DEFAULT_TAB_LABELS.map((label, i) => ({ label, settingKeys: SETTING_GROUP_KEYS[i] }));
  ```
- Store `resolvedTabs` as `this._resolvedTabs` for use by `_renderStructuredContent()`
- Pass `resolvedTabs.map(t => t.label)` to existing `createTabBar()` call (keep widget-based path working)
- Update fallback (non-widget) tab loop to use `resolvedTabs` for labels

Update `_renderStructuredContent()`:
- Replace `const groupKeys = SETTING_GROUP_KEYS[this._activeTab];` with:
  ```js
  const groupKeys = this._resolvedTabs?.[this._activeTab]?.settingKeys
    || SETTING_GROUP_KEYS[this._activeTab];
  ```
</action>

### T3: Update fallback tab rendering in `_renderStructured()` for icons

<read_first>
- src/ui/SettingsScreen.js (lines 472-495 — fallback tab loop)
- .planning/phases/53-configurable-tabs-engine/53-CONTEXT.md (D-05 through D-08)
</read_first>

<action>
In the fallback (non-widget) tab rendering section of `_renderStructured()`, update the loop to use `resolvedTabs` and support icons:

Replace the existing fallback loop that uses `tabLabels.forEach((label, i) => {...})` with one that iterates `resolvedTabs`:

```js
resolvedTabs.forEach((tab, i) => {
  const btn = document.createElement('button');
  btn.className = `settings-tab-btn${i === this._activeTab ? ' active' : ''}`;

  if (tab.icon) {
    btn.style.display = 'flex';
    btn.style.alignItems = 'center';
    btn.style.gap = '6px';
    const img = document.createElement('img');
    img.src = resolvePath(tab.icon);
    img.width = 24;
    img.height = 24;
    img.style.objectFit = 'contain';
    img.alt = '';
    btn.appendChild(img);
    const span = document.createElement('span');
    span.textContent = tab.label;
    btn.appendChild(span);
  } else {
    btn.textContent = tab.label;
  }

  btn.style.background = 'none';
  btn.style.border = 'none';
  btn.style.color = i === this._activeTab ? '#fff' : 'rgba(255,255,255,0.5)';
  btn.style.cursor = 'pointer';
  btn.style.padding = '8px 20px';
  btn.style.fontSize = '16px';
  btn.addEventListener('click', () => {
    this._activeTab = i;
    tabContainer.querySelectorAll('.settings-tab-btn').forEach((b, idx) => {
      b.classList.toggle('active', idx === i);
      b.style.color = idx === i ? '#fff' : 'rgba(255,255,255,0.5)';
    });
    this._renderStructuredContent(layout);
  });
  tabContainer.appendChild(btn);
});
```

Also update the widget-based path to pass `resolvedTabs` (with label+icon) to `createTabBar()` instead of just labels.
</action>
