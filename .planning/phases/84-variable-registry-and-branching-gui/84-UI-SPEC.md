## Phase 84 Source Lock

- `script.json.systems.variables` is the only project-level variable registry source of truth.
- Choice-side writes must save as canonical `effects[]`; Phase 84 UI may read legacy compat data but must not author raw-key writes as first-class saved output.
- Variable types stay limited to `bool` and `number`.
- Condition GUI is limited to 1-3 comparisons, operators `== != > >= < <=`, and mode `全部满足 / 任一满足`.
- Scene routing stores scene IDs internally but displays scene names everywhere user-facing.
- Rename/delete flows must expose references before applying destructive changes.
- Condition pages must converge on canonical saved fields: `conditionMode`, `conditions[]`, `trueTarget`, `falseTarget`.
- Each `conditions[]` row stores `{ variableId, operator, value }`.
- Legacy single-condition fields (`variable`, `operator`, `value`) may be read for compatibility, but Phase 84 save paths must emit the canonical shape.

## Feature Contract

### 1. Project-level variable registry UI

- Surface: `剧情系统` tab, not `项目设置`, not a modal.
- Layout:
  - header row with `📊 变量注册表` title, search field, type filter, group filter, and primary CTA `+ 新建变量`
  - left list shows: display name, muted internal ID, type badge, default value, usage count
  - right detail pane shows: display name, internal ID, type, default value, group, notes, reference list
- Row visuals:
  - row padding: 8px 16px
  - row radius: 6px
  - selected row background: `#094771`
  - selected row border: `#007acc`
- Type badges:
  - `布尔` uses neutral border/text styling
  - `数值` uses neutral border/text styling
  - no accent-colored type system; accent stays for selection only
- Empty state:
  - heading `还没有变量`
  - body `先创建第一个变量，然后在选项或条件页中引用它。`
  - primary CTA button centered under body
- Search/filter behavior:
  - search matches display name, internal ID, and group
  - filter changes do not clear the selected item if it still matches
  - no-result state copy: `没有匹配的变量`

### 2. Variable creation and editing rules

- `+ 新建变量` creates a draft row and immediately focuses display name in the detail pane.
- Required fields: display name, internal ID, type.
- Optional fields: group, notes.
- Default value control is type-aware:
  - `bool` → segmented true/false toggle labeled `是 / 否`
  - `number` → numeric input with step `1`
- Internal ID behavior:
  - auto-generated from display name on first create
  - manual edits are allowed
  - once the variable already has references, changing the ID must go through confirmation because it is a reference rewrite, not a silent text edit

### 3. Variable picker UX in page/choice editing

- Replace free-text variable key fields with `VariablePickerField`.
- Use the same picker in:
  - choice option effect rows
  - condition page comparison rows
- Picker behavior:
  - anchor to the current input row like the existing speaker dropdown pattern
  - first row is a search field
  - result rows show `显示名` on line 1 and muted `internalId · type · group` on line 2
  - selected row uses the standard `#094771` + `#007acc` selection treatment
  - footer action: `+ 新建变量`
- Missing data state:
  - if no variables exist, the picker body shows the standard variable empty state and the only primary action is `新建变量`
  - if a previously saved reference points to a missing variable, show inline warning `变量已不存在` plus secondary action `前往变量表`

### 4. Choice page effect editing contract

- Choice pages stop exposing `设置变量 = 值` compat inputs.
- Each option card gets an `效果` subsection under `跳转场景`.
- Each effect row contains:
  - action select: `设为 / 增加 / 减少`
  - variable picker
  - value control
- Value control rules:
  - `bool` variables only allow `设为`
  - `bool` value uses `是 / 否` segmented toggle
  - `number` variables allow `设为 / 增加 / 减少`
  - `number` value uses a numeric input
- Row actions:
  - `+ 添加效果` appends another row
  - row delete icon is always visible and must include text or title fallback `删除效果`
  - at least 0 effects is valid; empty `effects[]` means no write
- Option summary line under each card:
  - format: `{选项文本或“未命名选项”} · {效果摘要或“无变量变化”} · {跳转摘要}`
  - example: `答应她 · 樱好感 +1 · 跳转到 樱线入口`

### 5. Condition page GUI

- `SceneTree` must allow creating and converting to `条件页`; Phase 84 cannot leave condition pages as a hidden data-only type.
- `PageInspector` for `page.type === 'condition'` replaces dialogue editing with a dedicated condition builder.
- Builder sections, in order:
  1. summary block
  2. condition mode (`全部满足` / `任一满足`)
  3. 1-3 condition rows
  4. success target picker
  5. failure target picker
- Condition row contract:
  - variable picker first
  - operator second
  - value control third
  - delete-row affordance disabled when only 1 row remains and must include text or title fallback `删除条件`
  - `+ 添加条件` disabled after 3 rows
- Operator rules:
  - `bool` variables: `==`, `!=`
  - `number` variables: `==`, `!=`, `>`, `>=`, `<`, `<=`
- Target rules:
  - `满足时` target is required
  - `不满足时` target is optional; empty means `继续下一页`
- Summary format:
  - `若 {条件1} {且/或} {条件2...} → {成功场景}，否则 → {失败场景或继续下一页}`
  - always use variable display names and scene names in the summary
- Canonical save shape for UI/implementation alignment:
  ```json
  {
    "type": "condition",
    "conditionMode": "all",
    "conditions": [
      { "variableId": "route_sakura", "operator": "==", "value": true }
    ],
    "trueTarget": "scene_sakura",
    "falseTarget": null
  }
  ```

### 6. Scene picker + readable branch summary

- Use `SceneTargetPicker` in both choice option targets and condition page targets.
- Picker label uses scene name as primary text; show scene ID as muted helper text below the field after selection.
- If the target scene is missing, show inline error text: `目标场景不存在，请重新选择。`
- SceneTree summaries:
  - choice page row tooltip shows per-option branch summaries
  - condition page row text uses the condition summary, truncated to one line in the tree
- Inspector summaries:
  - render as a bordered summary box above detailed controls
  - summary box background: `#2a2a2a`
  - left accent border: `2px solid #007acc`
  - padding: `8px 16px`

### 7. Rename/delete safety affordances

- Every variable row shows a clickable usage badge: `{N} 引用`.
- Clicking the badge or opening the delete flow reveals reverse references grouped as:
  - `场景 > 页面 > 选项`
  - `场景 > 页面 > 条件`
- Rename flow:
  - display-name edits save inline with no modal
  - internal-ID edits with `N > 0` open `VariableImpactModal`
  - modal copy must explicitly say references will be rewritten, not broken
- Delete flow:
  - if `N = 0`, show a simple destructive confirm dialog
  - if `N > 0`, show reverse references before confirm
  - destructive CTA label: `删除变量并清除引用`
  - default focused action is always `取消`
- Post-delete behavior:
  - any cleared choice effect rows are removed, not left as broken placeholders
  - any cleared condition rows become unresolved warnings and require author repair before save if that would leave the page with zero valid rows

## Interaction States

- Loading state is not required; registry data is already inside loaded project state.
- Required empty states:
  - no variables in registry → `还没有变量` / `先创建第一个变量，然后在选项或条件页中引用它。` / primary action `新建变量`
  - no search results → `没有匹配的变量` / `试试其他关键字，或清除筛选条件。` / secondary action `清除筛选`
  - no effects on a choice option → `还没有效果` / `点击“添加效果”配置变量变化。` / primary action `添加效果`
  - missing variable reference in picker-backed rows → `变量已不存在` / `这个引用已经失效，请重新选择变量或前往变量表修复。` / actions `重新选择变量` + `前往变量表`
- Required error states:
  - duplicate variable ID → `变量 ID 已存在，请改用未占用的 ID。` / recovery action `继续编辑`
  - deleted/missing variable reference → `变量引用已失效，请重新选择变量。` / recovery action `重新选择变量`
  - deleted/missing scene target → `目标场景不存在，请重新选择。` / recovery action `选择目标场景`
  - invalid condition row because operator/value no longer matches variable type → `当前条件与变量类型不匹配，请重新选择运算符或输入值。` / recovery action `修复条件`

## Implementation Guardrails

- Preserve current dark VS Code-like editor tone; do not introduce light panels, glassmorphism, or token-heavy redesign.
- Stay within existing custom CSS + Vue SFC patterns; no Tailwind, no shadcn, no third-party registry blocks.
- Keep labels and helper copy in Chinese.
- Reuse current control styling conventions:
  - 6px card/input radius
  - 12-14px compact desktop text
  - blue hover/focus states
  - destructive hover stays red only on destructive actions
- Read legacy compat data if present, but all save paths must end in canonical Phase 83 contracts.
