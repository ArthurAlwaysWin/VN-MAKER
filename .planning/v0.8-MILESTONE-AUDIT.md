### Integration Gap: CUSTOM-03 Resolution Threading

**Severity:** Minor (default case works; non-default silently uses 1280×720)

**Root Cause:** ExportModal.vue `startExport()` does not extract `resolution` from `project.projectData` and does not pass `gameWidth`/`gameHeight` to the `export-game-desktop` IPC call.

**What works:** The entire pipeline supports custom resolution:
- `exportDesktop.js` accepts `gameWidth`/`gameHeight` parameters
- `game-main.js` template has `GAME_WIDTH`/`GAME_HEIGHT` placeholders
- Template filling replaces placeholders correctly

**What's missing:** ExportModal → IPC call doesn't include resolution fields.

**Fix (2 additions):**
```javascript
// ExportModal.vue startExport() — add to desktop IPC params:
gameWidth: project.projectData?.resolution?.width || 1280,
gameHeight: project.projectData?.resolution?.height || 720,
```
