# Brainstorm v2: 开源 Galgame Maker（视觉小说制作工具）

> 定位：**类似 RPG Maker，但面向视觉小说/Galgame 制作**。
> 非程序员也能通过可视化编辑器制作完整的视觉小说游戏。

---

## Goal

打造一个 **开源的 Galgame Maker**——一套完整的视觉小说制作工具，包含：
1. **Web 可视化编辑器**（Maker）：拖拽式/图形化界面，让不懂编程的创作者也能制作 galgame
2. **运行时引擎**（Runtime）：解析项目数据并运行游戏
3. **导出为独立游戏**：将完成的作品打包为可分发的桌面应用窗口

## Constraints

| 类别 | 约束 |
|------|------|
| **编辑器** | 基于 Web 技术构建，浏览器中即可使用 |
| **运行时** | 纯前端引擎，无后端依赖 |
| **导出** | 能打包为独立桌面窗口应用（Electron / Tauri / NW.js） |
| **用户群** | 核心用户是**非程序员**创作者，必须低门槛 |
| **开源许可** | MIT 或类似宽松许可 |
| **目标平台** | 仅 Windows 10 / 11，暂不考虑 macOS、Linux、移动端 |
| **暂缓功能** | CG 鉴赏、音乐鉴赏 → 后续迭代 |

## Known Context

- 项目目录当前为空白新项目
- 类似产品参考：
  - **RPG Maker**：最知名的游戏制作工具，可视化编辑 + 导出独立游戏（商业）
  - **Ren'Py**：最成熟的 VN 引擎，但需要写 Python 脚本，无 GUI 编辑器
  - **Tyrano Builder**：有可视化编辑器的 Web VN 引擎（日系商业）
  - **Visual Novel Maker** (by Degica)：Steam 上的 VN 制作工具（已停更）
  - **Monogatari**：纯 Web 开源 VN 引擎，写 JS 配置，无可视化编辑器

**市场空白**：目前没有一个**开源 + 可视化编辑器 + 可导出独立游戏**的 VN 制作工具。

## Risks

| 风险 | 影响 | 缓解 |
|------|------|------|
| **范围过大** | 编辑器 + 引擎 + 导出 = 三个系统 | 分阶段：先引擎 → 再编辑器 → 最后导出 |
| **编辑器复杂度高** | 可视化编辑器本身就是一个大型 Web 应用 | 先做最小可用的时间线/节点式编辑器 |
| **导出打包兼容性** | Electron/Tauri 打包涉及多平台构建 | 先支持 Windows，再扩展 Mac/Linux |
| **非程序员的 UX 设计** | 如果 UI 不够直觉，用户无法上手 | 参考 RPG Maker / Tyrano Builder 的交互范式 |
| **资源管理** | 图片/音频文件大，导出包体积膨胀 | 支持图片压缩、音频转码等优化 |

## Options

### Option A: 先做引擎 + 脚本，再补编辑器
- **Summary**: 先实现完整的运行时引擎（用 JSON/YAML 脚本驱动），引擎稳定后再开发可视化编辑器作为脚本的 GUI 前端
- **Pros**: 引擎与编辑器解耦，架构清晰；引擎本身对程序员也有价值；编辑器不阻塞引擎开发
- **Cons**: 编辑器到位前，非程序员无法使用
- **Complexity**: 中 | **Risk**: 低

### Option B: 编辑器优先，引擎同步开发
- **Summary**: 从编辑器 UI 开始设计，引擎作为编辑器的预览/运行模块同步开发
- **Pros**: 从一开始就保证非程序员友好；UI 驱动设计
- **Cons**: 编辑器和引擎耦合度高；前期开发量大；不容易独立测试引擎
- **Complexity**: 高 | **Risk**: 中

### Option C: 节点式脚本编辑器（类 Unreal 蓝图）
- **Summary**: 用节点/流程图式编辑器（如 React Flow）让用户连线编排剧情流程
- **Pros**: 分支剧情天然适合节点图表达；直觉且强大
- **Cons**: 节点编辑器本身开发量不小；长线性叙事时节点图不如时间线直观
- **Complexity**: 中高 | **Risk**: 中

### Option D: 时间线式编辑器（类视频剪辑软件）
- **Summary**: 采用时间线/轨道式编辑器，每条轨道对应一个层（文字、立绘、背景、音频），用户像剪辑视频一样编排内容
- **Pros**: 线性叙事直觉；轨道概念与 VN 多层系统天然匹配
- **Cons**: 复杂分支时不如节点图清晰；需要结合分支管理器
- **Complexity**: 中 | **Risk**: 低

## Recommendation

**推荐 Option A（先引擎后编辑器）+ 编辑器阶段采用 Option D（时间线）与 Option C（节点图）混合**

### 理由：
1. **先做引擎** 保证核心运行时稳定，定义好数据格式（JSON/YAML），这个数据格式就是编辑器与引擎之间的"契约"
2. 引擎本身就有独立价值——程序员可以直接用脚本制作游戏
3. **编辑器阶段** 结合两种模式：
   - **场景编辑器**（时间线式）：编排单个场景内的对话、立绘、背景、音频
   - **剧情流程编辑器**（节点图式）：管理场景之间的分支跳转和条件逻辑
4. **导出** 用 Electron/Tauri 将引擎 + 游戏资源打包为独立桌面应用

### 关于难度评估：
- **整体难度：中高**，但通过分阶段推进完全可行
- **Phase 1（引擎 MVP）**：约 2-4 周，一个人可完成
- **Phase 2（基础编辑器）**：约 4-8 周，需要较多 UI 工作
- **Phase 3（导出打包）**：约 1-2 周，Electron/Tauri 有成熟方案
- Web→独立窗口的导出难度**不大**，因为 Electron/Tauri 的核心就是把 Web 应用包成桌面程序

## Acceptance Criteria（MVP 阶段划分）

### Phase 1: Runtime Engine（运行时引擎）
- [ ] 定义游戏项目的数据格式规范（JSON/YAML）
- [ ] 文本显示（打字机效果、Auto、Skip、Backlog）
- [ ] 角色立绘系统（多角色、表情差分、进出场动画）
- [ ] 背景切换 + 转场特效
- [ ] 音频系统（BGM、SE）
- [ ] 选项 + 分支 + 变量系统
- [ ] 存档/读档系统
- [ ] 系统设置 + 主菜单
- [ ] 能运行一个完整的示例游戏

### Phase 2: Visual Editor（可视化编辑器）
- [ ] 场景编辑器（时间线式，编排对话/立绘/背景/音频）
- [ ] 剧情流程编辑器（节点图式，管理场景分支）
- [ ] 资源管理器（导入/管理图片、音频、字体）
- [ ] 实时预览（在编辑器中即时预览游戏效果）
- [ ] 项目保存/加载

### Phase 3: Export（导出打包）
- [ ] 导出为独立桌面应用（Electron 或 Tauri）
- [ ] Windows 平台优先
- [ ] 支持自定义窗口标题、图标、分辨率

### Deferred（暂缓）
- CG 鉴赏模式
- 音乐鉴赏模式
- 多语言 / i18n 支持
- 移动端适配
- macOS / Linux 支持与导出
- 插件/扩展系统
