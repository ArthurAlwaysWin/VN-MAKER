## Bug 修复

### Bug 1 — `captureGameScreenshot` 截图后对话框状态错乱

**文件：** `src/main.js`  
**严重程度：** 中等

**问题：** `DialogueBox.hide()` 只移除 `'visible'` 类，从不添加 `'hidden'` 类。截图函数却用 `classList.contains('hidden')` 来判断对话框是否可见，导致结果恒为 `true`，截图结束后对话框总会被强制设为可见——即使此时正处于选项菜单界面（对话框应当隐藏）。

**修复：** 将可见性判断改为检查 `'visible'` 类：

```diff
- const dlgWasVisible = !dialogueBox.el?.classList.contains('hidden');
+ // DialogueBox 只使用 'visible' 类 — 从不添加 'hidden'
+ const dlgWasVisible = dialogueBox.el?.classList.contains('visible');
```

### Bug 2 — `AudioManager.setBgmVolume` 调整主音量时忽略单轨音量

**文件：** `src/engine/AudioManager.js`  
**严重程度：** 中等

**问题：** `playBgm` 以 `data.volume * bgmVolume` 计算实际音量，但 `setBgmVolume` 调整主音量时直接赋值 `this._bgm.volume = vol`，忽略了当前 BGM 的单轨 `data.volume`。例如 BGM 以 `volume: 0.3` 播放，主音量调为 `0.8` 后应为 `0.24`，实际却变为 `0.8`。

**修复：** 新增 `_currentBgmTrackVolume` 字段，在 `playBgm` 时记录，`setBgmVolume` 时乘以此值：

```diff
+ this._currentBgmTrackVolume = 1; // 构造函数中初始化

  playBgm(data) {
+   this._currentBgmTrackVolume = data.volume ?? 0.5;
    const targetVolume = this._currentBgmTrackVolume * this.bgmVolume;
    ...
  }

  setBgmVolume(vol) {
    this.bgmVolume = vol;
    if (this._bgm) {
-     this._bgm.volume = vol;
+     this._bgm.volume = Math.max(0, Math.min(1, vol * this._currentBgmTrackVolume));
    }
  }
```

### Bug 3 — ESC 键无法打开游戏菜单

**文件：** `src/main.js`  
**严重程度：** 中等（功能缺失）

**问题：** ESC 键处理逻辑只能关闭各面板，当所有面板都已关闭时 ESC 什么也不做。`GameMenu` 实际上无法通过键盘打开，违反视觉小说类游戏的标准操作习惯。

**修复：** 在 ESC 关闭链末尾补充打开菜单逻辑：

```diff
  if (e.key === 'Escape') {
    if (!saveLoadScreen.el.classList.contains('hidden')) { ... return; }
    if (settingsScreen.isVisible) { ... return; }
    if (!backlogScreen.el.classList.contains('hidden')) { ... return; }
    if (!gameMenu.el.classList.contains('hidden')) { gameMenu.hide(); return; }
+   // ESC 无面板打开时 → 切换游戏菜单（标准视觉小说行为）
+   if (isPlaying) { gameMenu.show(); return; }
  }
```

### Bug 5 — `startAutoTimer` 创建的 Promise 回调无法取消

**文件：** `src/main.js`  
**严重程度：** 低（极端情况下可致 auto 关闭后仍推进对话）

**问题：** `Promise.all(waits).then(...)` 链不可取消。调用 `stopAuto()` 或快速切换 auto 模式时，`clearAutoTimer` 只清除了 interval/timeout，但已挂起的 Promise 回调仍可能触发 `engine.next()`。

**修复：** 引入 `_autoCallbackId` 世代计数器，回调执行前校验是否已过期：

```diff
+ let _autoCallbackId = 0;

  function startAutoTimer() {
    clearAutoTimer();
+   const myCallbackId = ++_autoCallbackId;
    ...
    Promise.all(waits).then(() => {
+     if (myCallbackId !== _autoCallbackId) return; // 过期回调，跳过
      if (autoMode && engine.waiting) engine.next();
    });
  }

  function clearAutoTimer() {
+   _autoCallbackId++; // 使所有挂起的回调失效
    ...
  }
```

## 代码质量优化

### 优化 1 — `ScriptEngine._execCondition` 遇到未知运算符时发出错误事件

**文件：** `src/engine/ScriptEngine.js`

未知运算符现在除了 `console.warn` 外还会 emit `'error'` 事件，便于上层（如编辑器预览）捕获并展示错误提示：

```diff
  default:
    console.warn(`[ScriptEngine] Unknown operator: ${page.operator}`);
+   this.emit('error', { type: 'unknown_operator', operator: page.operator, page });
```

### 优化 2 — `ScriptEngine` 历史记录内存上限

**文件：** `src/engine/ScriptEngine.js`

存档时历史截断到 50 条，但内存中 `engine.history` 可无限增长。现在在每条对话入队后检查并截断至 200 条，防止极长游戏会话中内存累积：

```diff
  this.history.push({ ... });
+ // 防止内存中历史记录无限增长 (HIST-01)
+ if (this.history.length > 200) {
+   this.history = this.history.slice(-200);
+ }
```

### 优化 3 — `electron/main.js` 存档槽位编号合法性校验

**文件：** `electron/main.js`

`save-slot` IPC handler 现在在写文件前验证槽位编号在 1-108 范围内：

```diff
  ipcMain.handle('save-slot', async (event, { slot, ... }) => {
    if (!currentProjectPath) return { success: false, error: 'No project loaded' };
+   if (!Number.isInteger(slot) || slot < 1 || slot > 108) {
+     return { success: false, error: 'Invalid slot number' };
+   }
```

### 优化 4 — `atomicWrite` 写入失败时回滚备份

**文件：** `electron/main.js`

原实现在 `rename(tmp → dest)` 失败时，`.bak` 文件会永久残留且原文件丢失。现在失败时自动尝试将 `.bak` 恢复为原文件：

```diff
- await fs.rename(tmp, filePath);
- try { await fs.unlink(bak); } catch {}
+ try {
+   await fs.rename(tmp, filePath);
+ } catch (err) {
+   try { await fs.rename(bak, filePath); } catch {} // 回滚
+   try { await fs.unlink(tmp); } catch {}
+   throw err;
+ }
+ try { await fs.unlink(bak); } catch {}
```
