# v1.4 Cinematic Upgrade Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver v1.4 cinematic foundations: preset character animations, page-level camera effects, editor-side configuration/preview, and expanded page transitions without introducing a freeform animation language.

**Architecture:** Keep the page-based script model intact and add only enum-driven fields (`character.animation`, `page.camera`, expanded `page.transition.type`). Runtime work is split into focused units: `ScriptEngine` emits data contracts, `CharacterLayer` owns character motion, `CameraController` owns stage-wide effects, and `BackgroundLayer` owns transition rendering. Editor work stays on the existing iframe preview path so preview and runtime share the same behavior.

**Tech Stack:** Vue 3 Composition API, Pinia, pure JavaScript ES modules, Vitest (jsdom), Node test runner, iframe postMessage, CSS transitions/keyframes

**Spec:** `docs/superpowers/specs/2026-04-21-v1.4-cinematic-upgrade-design.md`
