# 角色表情去背景工具 — 设计规格

**日期**: 2026-03-30
**状态**: 待实现
**范围**: 角色表情导入流程中的纯色背景去除

## 问题

角色立绘需要透明背景才能正确叠加在游戏场景上。用户可能导入无透明通道的图片（JPG、RGB PNG），导致角色在场景中显示矩形底色。目前系统已能检测并警告，但无法帮用户解决问题。

## 方案

在编辑器内集成纯色背景去除工具。当检测到无透明背景的角色图片时，弹出模态对话框，用户点击取色 + 调整参数即可生成透明 PNG。纯前端 Canvas API 实现，零新依赖，图片不离开本地。

## 功能边界

### 包含

- 纯色背景去除（白/黑/绿/蓝等单一颜色）
- 点击图片取色指定背景色
- 容差滑块（0-100）控制去除范围
- 边缘柔化滑块（0-5px）平滑轮廓
- 棋盘格透明预览 + 实时更新
- 两个入口：导入时自动触发 + 表情右键菜单

### 不包含

- 渐变背景去除
- AI 智能抠图
- 多颜色批量去除
- 非角色类别的图片处理

### 触发流程

**入口 1 — 导入自动触发：**

1. 用户点击"+ 导入表情" → 选择图片
2. `import-assets` IPC 返回 `noAlpha: true` 标记
3. `CharacterEditor.vue` 检测到标记 → 显示 `BgRemovalModal`
4. 用户操作去背景 → Canvas 导出透明 PNG
5. 通过 IPC 覆盖 `assets/characters/` 中的原文件
6. 刷新表情缩略图

**入口 2 — 右键菜单：**

1. 右键点击表情缩略图 → 菜单增加"去背景"选项
2. 加载已有图片到 `BgRemovalModal`
3. 操作同上 → 覆盖原文件 → 刷新缩略图

### 核心算法

```
输入: ImageData pixels, 目标颜色 (tR, tG, tB), 容差 T, 柔化 F

对每个像素 (R, G, B, A):
  distance = √((R-tR)² + (G-tG)² + (B-tB)²)

  if distance ≤ T:
    newAlpha = 0  // 完全透明
  else if distance ≤ T + F * scaleFactor:
    // 柔化过渡区：线性插值
    ratio = (distance - T) / (F * scaleFactor)
    newAlpha = ratio * 255
  else:
    newAlpha = A  // 保持不变

输出: 修改后的 ImageData → Canvas.toBlob('image/png')
```

- 色差计算：RGB 欧几里得距离，最大值 ≈ 441（√(255²×3)）
- 容差范围 0-100 映射到色差 0-441
- 柔化：在阈值边缘创建 alpha 渐变过渡带
- 性能：`requestAnimationFrame` 节流，参数变化时延迟 50ms 重新处理

### 取色交互

- Canvas 监听 `click` 事件
- `event.offsetX/Y` → `getImageData(x, y, 1, 1)` → 获取 RGB
- 更新右侧颜色显示 + 立即触发预览更新

### 文件保存

- `Canvas.toBlob('image/png')` → `FileReader.readAsArrayBuffer` → IPC 发送
- 新 IPC handler `save-processed-image`：接收 `{ filePath, data }` → `fs.writeFile` 覆盖
- 路径安全：`isInsideProject()` 校验

## 约束

- 零新 npm 依赖（纯 Canvas API + 原生 Vue）
- 纯本地处理（图片不离开用户电脑）
- 暗色主题，中文界面
- 仅处理纯色背景，不支持渐变/复杂背景
